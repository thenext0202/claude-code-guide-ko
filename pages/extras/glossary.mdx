## 터미널·셸

### 터미널 (Terminal)
> 컴퓨터에게 마우스 대신 **글자로 말을 거는 창**.

검은 (또는 파란) 화면이 뜨고 깜빡이는 커서가 있는 그 창입니다. macOS의 "터미널", Windows의 "PowerShell", Linux의 "Terminal" 모두 같은 개념.

**Claude Code는 이 안에서 동작해요.** → [04 설치와 첫 실행](/start/install) 참고.

### 셸 (Shell)
> 터미널 창 안에서 **실제 명령어를 해석해서 실행해주는 프로그램**.

터미널은 무대, 셸은 그 위에 선 진짜 일꾼이라고 보면 돼요. 종류:
- **bash** — Mac/Linux 기본
- **zsh** — 최신 macOS 기본
- **PowerShell** — Windows 기본 (다른 셸과 명령어 달라요)
- **Git Bash** — Windows에서 bash를 쓰게 해주는 변형

→ [Windows 풀가이드](/extras/windows-setup#1-핵심-결정--어떤-터미널을-쓸까)

### CLI
> **C**ommand **L**ine **I**nterface — "명령어로 조작하는 인터페이스".

터미널에 글자 쳐서 프로그램 쓰는 방식. 마우스로 클릭하는 GUI(Graphical User Interface)의 반대말.

`claude` 도, `git` 도, `npm` 도 다 CLI 도구예요.

### 프롬프트 (Prompt)
> 터미널에서 **명령을 입력 받는 자리**. 보통 `$` 또는 `>` 모양.

```bash
user@macbook ~ $   ← 이 부분이 프롬프트
```

이게 뜨면 "명령 쳐도 돼" 라는 뜻. Claude Code 안에서는 `>` 가 떠요.

> ⚠️ "프롬프트"라는 단어는 AI 분야에서는 **AI에게 주는 지시문**(예: "한국어로 답해줘")을 의미하기도 합니다. 문맥으로 구별.

### PATH
> 컴퓨터가 **명령어를 찾을 때 뒤지는 폴더 목록**.

`claude` 라고 쳤을 때, 컴퓨터는 PATH에 등록된 폴더들을 차례로 뒤져서 `claude` 라는 실행 파일을 찾아요. PATH에 없으면 `command not found` 에러가 나죠.

설치할 때 "Add to PATH" 체크하라는 게 이거 때문이에요.

→ [환경 변수](#환경-변수-environment-variable) 항목 참고.

## 파일·폴더

### 디렉토리 (Directory)
> **폴더와 같은 말**. 영어권에서 더 자주 씀.

`디렉토리 = 폴더 = directory = folder` — 다 같은 의미입니다.

### 경로 (Path)
> 컴퓨터 안에서 **파일/폴더의 주소**.

```
C:\Users\daniel\Desktop\프로젝트\hello.txt   ← Windows 경로
/Users/daniel/Desktop/project/hello.txt     ← Mac/Linux 경로
```

위에 나오는 [PATH](#path)와는 **완전히 다른 개념**. 헷갈리지 마세요.

→ [터미널 기초 명령어](/extras/terminal-basics#경로-읽는-법) 참고.

### 홈 폴더 / 홈 디렉토리
> **내 사용자의 기본 폴더.** 터미널에서는 `~` 한 글자로 부름.

| OS | 홈 폴더 위치 |
|---|---|
| macOS | `/Users/내이름` |
| Linux/WSL | `/home/내이름` |
| Windows | `C:\Users\내이름` |

`cd ~` 라고 치면 어느 위치에 있든 바로 홈 폴더로 이동.

### 숨김 파일 (Dotfile)
> **점(`.`)으로 시작하는 파일/폴더.** 보통 설정 파일이에요.

예: `.env`, `.gitignore`, `.claude/`

탐색기/Finder에서는 기본으로 숨겨져 있어요. 터미널에서는 `ls -a` 로 보여집니다.

### 확장자
> 파일 이름 끝의 점 뒤 부분. 파일 종류를 알려줌.

`hello.txt` → `txt` 가 확장자 (텍스트 파일)
`script.js` → `js` (JavaScript)
`README.md` → `md` (Markdown)

Windows는 기본으로 확장자가 숨겨져 있어요. 보이게 켜두는 걸 권장 (탐색기 → 보기 → 파일 확장명 체크).

## 설치·도구

### Node.js
> 자바스크립트를 **컴퓨터에서 직접 돌리는 엔진**.

원래 자바스크립트는 웹 브라우저 안에서만 돌아갔는데, Node.js는 그걸 컴퓨터에서 일반 프로그램처럼 돌리게 해줍니다. **Claude Code도 Node.js 위에서 동작**해요.

→ [설치 가이드](/start/install#2단계--nodejs-설치)

### npm
> Node.js의 **패키지 매니저**(앱스토어 같은 것).

`npm install -g 패키지이름` 으로 어떤 도구든 설치할 수 있어요. Claude Code도 `npm` 으로 설치합니다.

> npm = Node Package Manager의 약자.

### 패키지 (Package)
> **재사용 가능한 코드 묶음**. 누가 만들어서 npm에 올려둔 도구.

`@anthropic-ai/claude-code` 도 npm에 올라가 있는 패키지 중 하나예요.

### 전역 설치 (Global Install)
> **컴퓨터 전체에서 쓸 수 있게** 깔기.

```bash
npm install -g 패키지이름
```

`-g` 옵션이 "전역(global)" 의미. CLI 도구는 보통 전역 설치합니다. (Claude Code도 전역 설치)

반대는 **로컬 설치**(`npm install 패키지이름`) — 현재 폴더에만 설치돼요.

### 버전 (Version)
> 프로그램이 **몇 번째 판인지 표시.**

`v20.10.0` 처럼 숫자 세 개로 표기해요.
- 첫 번째 숫자 (20): 큰 변화 / 호환성 깨짐
- 두 번째 (10): 새 기능 추가
- 세 번째 (0): 버그 수정

`node --version` 같은 명령으로 확인.

## Claude Code 전용

### 세션 (Session)
> `claude` 명령으로 **시작해서 종료할 때까지의 한 번의 대화 단위**.

세션이 끝나면 Claude는 그 대화를 잊어요. 그래서 세션 관리가 중요합니다.

→ [10 세션 관리](/core/sessions)

### 컨텍스트 (Context)
> Claude가 **지금 이 순간 알고 있는 모든 것**.

대화 내용 + 읽은 파일 + CLAUDE.md + 도구 결과 = 컨텍스트.

크기 한계가 있어서, 너무 많이 쌓이면 답변 품질이 떨어져요. (`/clear` 또는 `/compact` 로 다이어트)

→ [07 핵심 개념](/core/concepts#1-context-컨텍스트--ai의-작업-메모리)

### 토큰 (Token)
> AI가 **글자를 세는 단위**. 대충 단어 1개 ≈ 1.3 토큰, 한국어 1글자 ≈ 1.5~2 토큰.

요금제 한도, 비용이 다 토큰 단위. 컨텍스트가 크면 토큰을 많이 쓰는 셈.

→ [03 요금제](/start/pricing#토큰-이-뭔데-한도가-걸려요), [20 비용 절약](/advanced/cost)

### 도구 (Tool)
> Claude가 **컴퓨터에 손을 댈 수 있는 능력 하나하나**.

- `Read` — 파일 읽기
- `Write` — 파일 만들기
- `Edit` — 파일 수정
- `Bash` — 명령어 실행
- `WebFetch` — 웹 페이지 가져오기

Claude가 작업할 때 화면에 `(Read tool)` 같은 표시가 뜨는 그것.

### 권한 (Permission)
> "이 도구는 자동 허용", "이건 매번 물어봄" 같은 **규칙 시스템.**

위험한 작업(파일 삭제, 강제 푸시 등)은 항상 Yes/No로 물어봐요. 자주 쓰는 안전한 명령은 미리 허용해두면 편합니다.

→ [핵심 개념 — Permissions](/core/concepts#3-permissions-권한--안전벨트)

### MCP
> **M**odel **C**ontext **P**rotocol — Claude가 외부 서비스(GitHub, Slack, DB 등)와 대화하는 표준 콘센트.

MCP를 통해 Claude가 GitHub PR을 만들거나, Slack 메시지를 보내거나, DB를 조회할 수 있어요.

→ [15 MCP](/advanced/mcp)

### Hooks
> 도구 사용 **전/후에 자동 실행되는 짧은 스크립트**.

위험 명령 자동 차단, 작업 완료 알림, 자동 로깅 등에 씁니다.

→ [16 Hooks](/advanced/hooks)

### Skills
> 자주 하는 작업을 **`/단축어` 한 줄로 호출하게** 저장한 레시피.

매번 길게 부탁하는 대신 `/회의록정리` 같은 슬래시 명령 하나로 끝.

→ [17 Skills](/advanced/skills)

### 서브에이전트 (Sub-agent)
> 메인 Claude가 **별도 작업을 맡기는 또 다른 Claude.**

전문가를 따로 두는 셈. 검색은 Explore, 코드 리뷰는 code-reviewer, 식.

→ [18 Sub-agents](/advanced/subagents)

### CLAUDE.md
> 폴더 안의 **자동으로 읽히는 안내서.** Claude가 작업 시작 시 항상 읽음.

신입사원에게 주는 회사 매뉴얼 같은 것.

→ [08 CLAUDE.md](/core/claude-md)

### .claude 폴더
> Claude Code의 **설정·스킬·후크가 저장되는 숨김 폴더**.

```
~/.claude/                    ← 전역 설정 (모든 작업에 적용)
├── CLAUDE.md                ← 내 프로필
├── settings.json            ← 권한, 후크
├── skills/                  ← 내 스킬들
├── agents/                  ← 내 서브에이전트들
├── memory/                  ← 자동 메모리
└── projects/                ← 세션 로그

내프로젝트/.claude/            ← 프로젝트별 (이 폴더에서만 적용)
├── CLAUDE.md
├── settings.json
└── skills/
```

점(`.`)으로 시작해서 기본적으로 숨겨져 있어요. 탐색기/Finder에서 "숨김 파일 보기" 켜야 보입니다.

→ [에디터 셋업](/extras/editor-setup#claude-폴더-열어보기)

## 파일 형식

### JSON
> **J**ava**S**cript **O**bject **N**otation — 사람도 읽고 컴퓨터도 읽는 **데이터 표기법.**

```json
{
  "name": "Daniel",
  "age": 30,
  "languages": ["한국어", "English"]
}
```

규칙:
- `{}` 로 묶고
- 키와 값을 `:` 로 연결
- 항목 사이는 `,` 로 구분
- 마지막 항목 뒤엔 콤마 X
- 모든 키는 큰따옴표 `"`

`settings.json`, `mcp.json` 같은 파일이 이 형식.

> 💡 JSON 잘못 쓰면 Claude Code가 설정을 못 읽어요. 헷갈리면 [JSONLint](https://jsonlint.com)에 붙여넣어서 검증.

### Markdown / MDX
> **마크다운(Markdown)** = 글에 굵게, 기울임, 표, 코드 같은 서식을 **간단한 기호**로 적는 방식.

```markdown
# 제목
**굵게**, *기울임*
- 리스트 항목 1
- 리스트 항목 2
```

확장자는 `.md`. CLAUDE.md, README.md 등이 다 이 형식.

**MDX**는 마크다운 + JSX(React) 컴포넌트. 이 가이드 자체가 MDX로 짜여 있어요.

### YAML
> JSON 친척. **들여쓰기와 콜론**으로 데이터 표현.

```yaml
name: Daniel
age: 30
languages:
  - 한국어
  - English
```

CLAUDE.md나 SKILL.md의 frontmatter에 자주 등장.

### Frontmatter
> 마크다운 파일 **맨 위에 `---` 로 감싼 메타데이터** 영역.

```markdown
---
title: 04 · 설치와 첫 실행
description: 한 단계씩 천천히
---

# 본문 시작
...
```

사이의 내용은 YAML 형식. Skills, Sub-agents 정의에서 필수.

## Git·GitHub

### Git
> 코드/파일 **변경 이력을 시간 단위로 기록**해주는 도구. "시간 여행"이 가능해짐.

→ [11 GitHub와 함께 쓰기](/practice/github#왜-git이-필요해요)

### GitHub
> **Git으로 관리되는 코드를 인터넷에 올려두는 서비스.**

내 노트북에서 짠 코드를 GitHub에 올리면 다른 컴퓨터/동료와 공유 가능. 카카오톡이 메시지 저장 서비스라면, GitHub은 코드 저장 서비스.

> Git ≠ GitHub. Git은 도구, GitHub는 그 도구를 쓰는 클라우드 서비스. (GitLab, Bitbucket도 비슷한 서비스.)

### 레포지토리 (Repository, Repo)
> Git이 관리하는 **하나의 프로젝트 폴더 단위**.

"이 레포에 푸시해줘" = "이 프로젝트에 코드 올려줘".

### 커밋 (Commit)
> 변경 사항을 **저장(스냅샷)** 해두는 행위.

```bash
git commit -m "다크모드 추가"
```

이러면 그 시점의 폴더 상태가 영원히 기록돼요. 나중에 "이 시점으로 돌아가" 가능.

### 브랜치 (Branch)
> **평행 우주.** main 브랜치는 안전하게 두고, 새 작업은 별도 브랜치에서 시도.

```bash
git checkout -b feature/dark-mode    # 새 브랜치 만들고 이동
```

문제 없으면 main에 합치고(merge), 망하면 그 브랜치만 버리면 끝.

### 푸시 / 풀 (Push / Pull)
- **Push**: 내 컴퓨터의 커밋을 **GitHub에 올림.**
- **Pull**: GitHub의 변경사항을 **내 컴퓨터로 받음.**

### PR (Pull Request)
> "이 브랜치를 main에 합쳐줘" 라고 GitHub에서 **공식 요청**하는 것.

리뷰·토론·승인 후 merge. 협업의 기본 단위.

## 운영체제

### Windows
Microsoft의 OS. PC 시장 다수.

### WSL
> **W**indows **S**ubsystem for **L**inux — Windows 안에서 진짜 리눅스를 돌리는 가상환경.

`wsl --install` 한 번으로 깔립니다. 개발자에게 가장 추천되는 환경.

→ [Windows 풀가이드](/extras/windows-setup#5-옵션-c--wsl우분투로-시작하기--개발자-추천)

### PowerShell
> Windows 기본 셸. 별도 설치 없이 바로 사용 가능.

bash와 명령어 문법이 달라요. (`ls` 는 비슷하지만, 옵션이나 파이프가 다름)

### Git Bash
> Windows에서 bash 명령어를 쓰게 해주는 작은 터미널. Git을 깔면 같이 옴.

비개발자/초보자에게 가장 추천. 튜토리얼 호환성이 좋아요.

### macOS
> 애플 컴퓨터의 OS. 기본 셸은 zsh, 터미널은 "터미널" 앱.

### Linux
> 오픈소스 OS. Ubuntu, Debian, Fedora, Arch 등 여러 배포판이 있음.

서버 환경 / 개발자 데스크톱.

### 환경 변수 (Environment Variable)
> 컴퓨터 전체가 **공용으로 쓰는 변수**.

대표적으로 [PATH](#path)가 환경 변수의 일종. API 키처럼 자주 쓰는 비밀 값을 환경 변수에 두기도 해요.

```bash
# Mac/Linux/WSL/Git Bash
export GITHUB_TOKEN="ghp_xxxxx"

# Windows PowerShell
$env:GITHUB_TOKEN = "ghp_xxxxx"
```

→ [Windows 풀가이드 — 환경 변수](/extras/windows-setup#환경-변수-편집-클릭-단위)

### 관리자 권한
> Windows에서 **시스템 영역에 영향 줄 수 있는 작업**을 할 때 필요한 상승된 권한.

PowerShell을 "관리자 권한으로 실행" 하면 창 제목에 **"관리자:"** 가 붙어요. npm 글로벌 설치 같은 작업은 보통 이게 필요.

→ [PowerShell 관리자 권한 여는 법](/extras/windows-setup#powershell-여는-법)

## 기타

### API
> **A**pplication **P**rogramming **I**nterface — 한 프로그램이 다른 프로그램과 **대화하는 약속된 통로.**

OpenAI API = OpenAI에게 "이 질문 답해줘" 라고 보낼 수 있는 통로. Anthropic API = Claude에게 보내는 통로.

Claude Code는 내부적으로 Anthropic API를 사용하지만, **사용자는 따로 API 키 발급 안 해도** Pro/Max 구독으로 쓸 수 있어요.

### API 키
> API를 호출할 때 "이 사람 맞다" 인증하는 **비밀번호 같은 문자열**.

```
sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**절대 GitHub에 올리거나 코드에 직접 적지 마세요.** 한 번 노출되면 영원히 남고, 도용 시 막대한 비용이 청구될 수 있음.

### LTS
> **L**ong-**T**erm **S**upport — **장기 지원 버전.**

Node.js를 받을 때 "LTS" 라고 적힌 큰 초록 버튼을 누르라는 게 이거. 한동안 안정적으로 지원되는 버전이라 안전.

### UTF-8
> 전 세계 문자(한글, 한자, 이모지)를 **모두 표현할 수 있는 글자 인코딩 규칙.**

`chcp 65001` 명령은 "이 창에서 UTF-8 쓸게"라는 뜻. Windows에서 한글 깨짐 해결의 시작.

### 인코딩 (Encoding)
> 컴퓨터가 글자를 **숫자로 저장하는 방식.**

UTF-8, EUC-KR, CP949 등이 있는데, 서로 다른 인코딩으로 저장된 파일을 다른 인코딩으로 열면 한글이 깨져 보여요.

해결: 다 **UTF-8 (BOM 없음)** 으로 통일.

### 프록시 (Proxy)
> 회사 등에서 **인터넷 요청을 중간에서 검사하고 거르는 서버.**

회사 노트북에서 npm 설치가 안 되면 프록시 설정이 필요할 때가 있어요.

```bash
npm config set proxy http://proxy.company.com:8080
```

→ [Windows 풀가이드 — 회사 환경](/extras/windows-setup#9-회사-보안-환경에서)

### 백슬래시 (`\`) vs 슬래시 (`/`)
> Windows는 경로 구분자로 **`\`**, Mac/Linux는 **`/`** 를 씁니다.

```
C:\Users\daniel\Desktop      ← Windows
/Users/daniel/Desktop        ← Mac/Linux
```

Git Bash, WSL, PowerShell 7+ 에서는 `/` 도 보통 동작해요. 이 가이드는 통일성을 위해 대부분 `/` 로 적습니다.

> 💡 백슬래시는 한국어 키보드에서 `₩` 키와 같은 자리예요. 헷갈리지만 같은 글자입니다.
