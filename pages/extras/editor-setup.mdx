# 에디터 셋업 — 설정 파일 만질 준비

> Claude Code의 진짜 힘은 **설정 파일**(`settings.json`, `CLAUDE.md`, `SKILL.md` 등)에서 나와요. 이걸 편하게 만지려면 좋은 에디터가 필요합니다.
>
> 이 페이지는 **에디터를 써본 적 없다는 분 기준**으로 썼어요. 메모장도 됩니다 — 단, VS Code가 훨씬 편해요.

> 💡 같이 보면 좋은 페이지: [📖 용어집](/extras/glossary) · [⌨️ 터미널 기초](/extras/terminal-basics) · [04 설치와 첫 실행](/start/install)

---

## 한 눈에 — 이 페이지 끝나면 할 수 있는 것

- VS Code(또는 메모장)로 **`settings.json` / `CLAUDE.md`** 같은 파일 만들기·열기
- **`.claude/`** 같은 **숨김 폴더** 보고 들어가기
- 파일을 **UTF-8** 인코딩으로 저장해서 한글 안 깨지게 하기
- VS Code의 **Claude Code 통합** 기능 켜기

---

## 1. 어떤 에디터를 써야 해요?

### 빠른 결론

| 도구 | 추천도 | 한 줄 평 |
|---|---|---|
| **VS Code** | ⭐⭐⭐⭐⭐ | 무료·가볍고·Claude Code 통합 가능. **이 가이드 기본 추천** |
| **Cursor** | ⭐⭐⭐⭐ | VS Code 포크. AI 통합 강력하지만 Claude Code랑 겹침 |
| **메모장 (Notepad)** | ⭐⭐ | 진짜 급할 때 가능. 하지만 한글 인코딩 함정 있음 |
| **워드/한글** | ❌ | 절대 사용 금지. 보이지 않는 서식이 들어가서 파일 망가뜨림 |

> 💡 비개발자도 VS Code 하나는 깔아두는 게 좋아요. 평생 쓰게 됩니다.

## 2. VS Code 설치 (3분)

### 1단계 · 다운로드

1. 브라우저로 [code.visualstudio.com](https://code.visualstudio.com) 접속
2. 큰 파란 버튼 — 본인 OS에 맞춰 자동 안내됨
   - macOS: "Download Mac Universal"
   - Windows: "Download for Windows"
   - Linux: 패키지 (.deb / .rpm) 또는 snap
3. 다운로드 파일 더블클릭

### 2단계 · 설치

#### macOS
- 다운받은 `.zip` 압축 풀면 `Visual Studio Code.app` 이 나옴
- **응용 프로그램(Applications) 폴더로 드래그**

#### Windows
- 다운받은 `.exe` 더블클릭
- **체크 박스 추천 설정**:
  - ✅ "Add 'Open with Code' action to Windows Explorer file context menu" (폴더 우클릭에서 VS Code 열기)
  - ✅ "Add 'Open with Code' action to Windows Explorer directory context menu"
  - ✅ "Register Code as an editor for supported file types"
  - ✅ "Add to PATH" (터미널에서 `code` 명령 사용 가능)

#### Linux
```bash
sudo apt install code     # Ubuntu/Debian (microsoft repo 등록 후)
# 또는 snap
sudo snap install code --classic
```

### 3단계 · 한국어로 바꾸기 (선택)

VS Code는 기본 영어. 한국어 메뉴가 편하면:

1. VS Code 실행
2. 왼쪽 사이드바에서 **사각형 4개 아이콘** (Extensions / 확장)
3. 검색창에 `Korean Language Pack` 검색
4. Microsoft 제작 한국어 팩 → **Install**
5. 우하단에 "Restart" 알림 → 재시작
6. 메뉴가 한국어로 바뀜

> 💡 영어가 더 편한 경우도 있어요. 외국 자료 검색할 때 메뉴명이 영어면 매칭이 쉬움. 부담 없이 둘 다 시도해보세요.

### 4단계 · 터미널에서 `code` 명령 켜기 (Mac만)

Mac은 자동 등록이 안 돼서 한 번 설정 필요:

1. VS Code 안에서 `Cmd + Shift + P`
2. 나오는 검색창에 `shell command` 입력
3. **"Shell Command: Install 'code' command in PATH"** 클릭

이러면 터미널에서:
```bash
code .             # 현재 폴더를 VS Code로 열기
code hello.txt     # 특정 파일 VS Code로 열기
```

→ 한 번 해두면 평생 편함.

## 3. 첫 폴더 열기

### 방법 A — VS Code에서 메뉴로
1. VS Code 실행 → File → **Open Folder** (또는 `파일 → 폴더 열기`)
2. 작업할 폴더 선택 → Open
3. 왼쪽 사이드바에 폴더 트리가 떠요

### 방법 B — 터미널에서
```bash
cd ~/Desktop/my-claude-test
code .             # "지금 이 폴더를 VS Code로 열어"
```

`.` 이 "지금 이 폴더" 라는 뜻 ([터미널 기초](/extras/terminal-basics) 참고).

### 방법 C — 폴더 우클릭 (Windows/Mac 컨텍스트 메뉴 설정 시)
- 탐색기/Finder에서 폴더 우클릭 → **"Open with Code"**

## 4. 새 파일 만들기 — `settings.json` 예제

`.claude/settings.json` 만들어보면서 흐름 익히기.

### VS Code에서

1. 왼쪽 사이드바 폴더 트리에서 **빈 곳에 우클릭** → **New Folder**
2. 이름: `.claude` (점 포함, 영문, 띄어쓰기 X) → Enter
3. 만들어진 `.claude` 폴더에 우클릭 → **New File**
4. 이름: `settings.json` → Enter
5. 빈 파일이 열림. 다음 내용 붙여넣기:

```json
{
  "permissions": {
    "allow": [
      "Read(**)",
      "Bash(git status)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}
```

6. **Cmd/Ctrl + S** 로 저장

### JSON 문법이 맞는지 확인

저장 후 VS Code가 빨간 줄을 표시하면 문법 오류:
- 키와 값을 `:` 로 연결했나?
- 항목 사이를 `,` 로 구분했나?
- 마지막 항목 뒤에 `,` 안 찍었나?
- 모든 키가 큰따옴표 `"` 로 감싸졌나?

→ [용어집 — JSON](/extras/glossary#json)

## 5. `CLAUDE.md` 만들기

위와 똑같은 흐름:

1. 프로젝트 폴더 루트에 우클릭 → **New File**
2. 이름: `CLAUDE.md` (대소문자 정확히)
3. 다음 예시 붙여넣고 저장:

```markdown
# 이 프로젝트는?
한국 카페 추천 사이트. Next.js + Tailwind.

# 코드 스타일
- 한국어 주석
- 함수명은 영어 camelCase
- 큰 변경 전엔 git commit

# 절대 하지 마세요
- main 브랜치에 직접 푸시 금지
- .env 파일 수정 금지
```

이게 끝. 다음 `claude` 실행부터 자동으로 읽힙니다.

→ 자세한 작성법: [08 CLAUDE.md](/core/claude-md)

## 6. 숨김 파일·폴더 보기

`.claude/`, `.env`, `.gitignore` 같은 점(`.`)으로 시작하는 파일은 **기본적으로 숨겨져** 있어요.

### VS Code
사이드바 트리에는 **자동으로 보입니다.** (걱정 X)

### macOS Finder
- 키보드 `Cmd + Shift + .` (점) → 숨김 파일 토글

### Windows 탐색기
- 탐색기 상단 → **보기** → **표시** → **숨긴 항목** 체크
- (Windows 11) 탐색기 상단 ⋯ → **표시** → **숨긴 항목**

### Linux 파일 매니저
- `Ctrl + H` 토글

### 터미널에서
```bash
ls -a       # Mac/Linux/WSL/Git Bash
ls -Force   # PowerShell
```

## 7. UTF-8로 저장 (한글 깨짐 방지)

Windows 메모장에서 저장하면 가끔 인코딩이 ANSI / CP949 가 되어서 한글이 깨져요. **항상 UTF-8 (BOM 없음)** 으로 저장하세요.

### VS Code 인코딩 확인·바꾸기
1. 파일 열린 상태에서 **우하단 상태바**를 보세요
2. `UTF-8` 이라고 적힌 글자 클릭
3. **"Save with Encoding"** → `UTF-8` 선택
4. 한 번만 하면 그 파일은 계속 UTF-8

### 메모장에서 저장 시 (어쩔 수 없을 때)
1. 다른 이름으로 저장
2. 인코딩 드롭다운에서 **UTF-8** 선택 (BOM 있는 UTF-8은 피하기)

## 8. `.claude/` 폴더 열어보기

이 폴더가 어디에 있는지 + 안에 뭐가 들어있는지 한 번 들여다보면 자신감 폭증.

### 위치

| OS | 전역 .claude 위치 |
|---|---|
| macOS | `~/.claude/` (= `/Users/내이름/.claude/`) |
| Linux / WSL | `~/.claude/` (= `/home/내이름/.claude/`) |
| Windows | `C:\Users\내이름\.claude\` |

프로젝트별은 **그 프로젝트 폴더 안의 `.claude/`** 에 있어요.

### 직접 가서 보기

```bash
# Mac/Linux/WSL/Git Bash
cd ~/.claude
ls -la

# Windows PowerShell
cd $env:USERPROFILE\.claude
ls -Force
```

### 보일 만한 것

```
~/.claude/
├── CLAUDE.md           ← 내 전역 프로필 (있을 수도/없을 수도)
├── settings.json       ← 권한·후크 설정
├── skills/             ← 내가 만든 Skills
├── agents/             ← 내가 만든 서브에이전트
├── memory/             ← 자동 메모리 (만들어지면 자동 생성)
├── mcp.json            ← MCP 서버 설정
├── projects/           ← 세션 로그 (자동 생성)
└── ...
```

→ 각 항목의 의미: [용어집 — .claude 폴더](/extras/glossary#claude-폴더)

### VS Code에서 통째로 열기

```bash
code ~/.claude          # Mac/Linux/WSL/Git Bash
code $env:USERPROFILE\.claude    # PowerShell
```

이러면 VS Code에 `.claude` 폴더가 열리고, 모든 설정 파일을 한눈에 볼 수 있어요.

## 9. VS Code + Claude Code 통합 (`/ide`)

VS Code 안에서 **터미널 + Claude Code + 코드 편집을 한 화면**에 띄울 수 있어요.

### 설치

VS Code 확장 마켓플레이스에서 검색:
- "Claude Code" (Anthropic 공식 확장이 있다면)
- 또는 그냥 VS Code 내장 터미널을 쓰면서 `/ide` 명령

### 사용 흐름

1. VS Code에서 작업 폴더 열기
2. 터미널 패널 열기: `Ctrl + ` ` ` (백틱) — 또는 메뉴 → 터미널 → 새 터미널
3. 터미널에서 `claude` 실행
4. Claude 안에서 `/ide` 입력 → IDE 통합 모드 ON

이러면:
- 왼쪽: 코드 트리
- 가운데: 코드 편집
- 아래: Claude 대화창

같은 화면에서 **Claude가 어떤 파일을 열었는지 실시간으로 보임.**

## 10. 자주 헷갈리는 점

### "파일 이름 끝에 점이 있는데 맞나요?"

`settings.json`, `CLAUDE.md` 같은 이름의 **마지막 점**은 확장자를 알리는 거지, 점으로 시작하는 게 아니에요.

| 파일 이름 | 종류 |
|---|---|
| `settings.json` | 일반 파일, 확장자가 .json |
| `.gitignore` | 숨김 파일 (점으로 시작) |
| `.claude/settings.json` | 숨김 폴더 안의 일반 파일 |

### "메모장으로 settings.json 저장했는데 settings.json.txt 가 됐어요"

Windows 메모장의 함정. 저장 시:
- 파일 형식: **모든 파일 (`*.*`)** 로 변경
- 파일 이름: `settings.json` (따옴표 X, 정확히)
- 인코딩: **UTF-8** (BOM 없음)

해결되면 **VS Code로 갈아타세요.** 메모장은 함정이 너무 많음.

### ".claude 폴더가 어디 있는지 안 보여요"

탐색기/Finder의 **숨김 파일 보기**가 꺼져 있어서. 위 7번 섹션 참고.

또는 그냥 터미널에서:
```bash
code ~/.claude     # 무조건 열림
```

### "VS Code 켰는데 한글 폰트가 못생겼어요"

설정 (`Cmd/Ctrl + ,`) → `Editor: Font Family` 검색 → 다음 중 하나로:
```
'D2Coding', 'Pretendard', 'JetBrains Mono', monospace
```

### "코드/JSON에 자동 색칠 안 돼요"

파일 우하단 → 언어 모드 클릭 → 적절한 언어 선택. 또는 확장자가 맞는지 확인 (`.json`, `.md`, `.js` 등).

## 12. 응급 — 에디터 없이 터미널에서 파일 편집

설치할 시간도 없을 때:

### nano (가장 쉬움)
```bash
nano settings.json
```
- 화면 아래에 단축키 안내
- `Ctrl + O` 저장, `Ctrl + X` 종료

### vim (강력하지만 처음엔 어려움)
```bash
vim settings.json
```
- 처음 들어가면 **Esc → `i` → 편집 → Esc → `:wq` Enter** 로 저장+종료
- 모르고 들어가면 못 빠져나오는 게 함정. **`Esc` → `:q!` Enter** 로 강제 종료.

> 💡 처음엔 nano만 알아도 충분. vim은 나중에.
