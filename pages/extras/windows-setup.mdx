# Windows 환경 풀 셋업 가이드

> 한국 사용자가 첫 1주일에 막히는 90%는 **Windows 환경** 때문입니다. 이 페이지는 **"터미널이 뭔지 처음 듣는 분"** 을 기준으로 썼어요. 천천히 따라오세요.

## 0. 잠깐, 터미널이 뭐예요?

평소엔 마우스로 폴더를 더블클릭해서 파일을 열죠. **터미널은 그걸 글자로 하는 도구**예요.

```
"바탕화면 폴더로 가서 hello.txt 파일을 만들어"
```

이걸 마우스 대신 키보드로 한 줄 치는 곳입니다. 검은 (또는 파란) 화면에 글자만 나와서 처음엔 어색한데, **Claude Code가 살아 움직이는 무대가 바로 이 터미널**이에요.

Windows에는 터미널이 여러 종류 있어요. 그 차이가 이 페이지의 핵심입니다.

> 📚 **같이 보면 좋은 페이지**
> - [📖 용어집](/extras/glossary) — 모르는 단어 사전
> - [⌨️ 터미널 기초 명령어](/extras/terminal-basics) — `cd`, `ls`, `pwd` 등 사용법
> - [✏️ 에디터 셋업](/extras/editor-setup) — VS Code 설치 + 설정 파일 만지기

## 1. 핵심 결정 — 어떤 터미널을 쓸까?

Windows에서 Claude Code를 돌릴 환경은 **3가지**입니다. 어느 걸 골라도 Claude Code 자체는 작동하지만, **편의성과 막힘 빈도가 크게 달라요.**

### 🪟 PowerShell

> Windows에 **이미 깔려있는** 기본 터미널. 추가 설치 0.

| 장점 | 단점 |
|---|---|
| 추가 설치 필요 없음 (지금 바로 사용) | 한글이 `????` 로 깨지는 경우가 자주 있음 |
| Windows 네이티브 (가장 빠름) | Mac/Linux 튜토리얼에 나오는 명령어 일부가 안 통함 (예: `cat`, `grep`, 파이프) |
| 회사 보안정책에 가장 안전 | bash 스크립트가 안 돌아감 |

**적합한 사람**: "일단 30초 안에 깔아보고 싶다" / 회사 정책상 다른 게 막혀 있다.

---

### 🟢 Git Bash

> Git을 깔면 같이 따라오는 **bash 호환 터미널**. 한 번만 설치하면 끝.

| 장점 | 단점 |
|---|---|
| Mac/Linux 명령어가 그대로 동작 (튜토리얼 그대로 따라할 수 있음) | 처음 한 번 설치 필요 (~300MB) |
| Git이 같이 깔려서 일석이조 | 일부 고급 도구·고급 옵션은 호환성 한계 |
| 한글 인코딩 문제가 PowerShell보다 적음 | npm 글로벌 권한 설정 한 번은 해줘야 함 |
| Windows 파일 시스템에 그대로 접근 (별도 경로 변환 X) | |

**적합한 사람**: **대부분의 비개발자·초보자**. 튜토리얼이나 가이드를 보고 따라하는 게 편한 분. **이 가이드의 기본 추천 환경입니다.**

---

### 🐧 WSL (Ubuntu)

> Windows 안에서 진짜 **리눅스를 돌리는** 가상환경. 가장 강력하지만 가장 무거움.

| 장점 | 단점 |
|---|---|
| Mac/Linux와 100% 동일한 환경 (호환성 최강) | 설치 절차가 가장 길다 (관리자 권한, 재부팅 필요) |
| 한글 인코딩 거의 문제없음 | 디스크 2GB+ 추가 필요 |
| 거의 모든 개발 도구·MCP 서버가 잘 작동 | Windows 폴더에 접근하려면 `/mnt/c/...` 거쳐야 하고 느림 |
| 장기적으로 본격 개발할 때 가장 유리 | 회사 노트북에서 막혀 있는 경우가 종종 있음 |

**적합한 사람**: 진지하게 코딩을 배우거나 개발자로 일할 분. "오래 쓸 것 같다"는 분.

---

### 한 줄 추천

```
"빨리 한 번 써보기만 하고 싶다"          → PowerShell
"Claude Code 가이드·튜토리얼 따라하기 편하게"  → Git Bash       ★ 비개발자 추천
"코딩 본격적으로 배울 거다 / 개발자다"      → WSL            ★ 개발자 추천
```

> 💡 결정 못 하겠다면 **Git Bash로 시작**하세요. 나중에 WSL로 갈아타도 늦지 않고, 그때 익힌 명령어가 그대로 통합니다. PowerShell만 익히면 나중에 다 다시 배워야 해요.

## 2. 터미널 여는 법 (클릭 단위 안내)

선택한 터미널을 여는 법부터 알려드릴게요.

### PowerShell 여는 법

**방법 1 (가장 쉬움)**
1. 키보드에서 **`Windows 키`** 한 번 누르기 (시작 메뉴 열림)
2. 그대로 키보드로 **`powershell`** 타이핑
3. 검색 결과에 "Windows PowerShell" 또는 "PowerShell" 보임 → **`Enter`**

**방법 2 — 관리자 권한으로 열기 (설치할 때 필요)**
1. `Windows 키` → `powershell` 입력
2. 검색 결과에서 **마우스 우클릭** → **"관리자 권한으로 실행"**
3. "이 앱이 디바이스를 변경할 수 있도록 허용하시겠어요?" → **"예"**

> 관리자 권한 창은 **창 제목 맨 위에 "관리자:"** 가 붙어 있어요. 그게 보이면 OK.

### Git Bash 여는 법 (설치 후)

1. `Windows 키` → `git bash` 입력 → `Enter`
2. 또는 아무 폴더에서 **빈 곳에 우클릭** → **"Git Bash Here"** (이게 정말 편해요)

### WSL(Ubuntu) 여는 법 (설치 후)

1. `Windows 키` → `ubuntu` 입력 → `Enter`
2. 또는 PowerShell에서 `wsl` 입력하고 Enter

## 3. 옵션 A · PowerShell로 시작하기 (추가 설치 0)

> 가장 빠르게 시작하는 길. 단, 한글이 깨지면 [4번 섹션](#4-한국어--인코딩-문제) 보세요.

### 1단계 · Node.js 설치

Node.js는 Claude Code가 돌아가는 기반입니다.

1. 브라우저로 [nodejs.org](https://nodejs.org) 접속
2. 큰 초록 버튼 **"LTS"** 클릭 → 설치 파일(`.msi`) 다운로드
3. 다운로드한 파일 더블클릭
4. **Next → Next → Next** 누르며 설치
   - "Add to PATH" 체크박스가 보이면 **반드시 체크** (보통 기본 체크됨)
5. 끝나면 **PowerShell 창을 한 번 닫고 새로 열기** (PATH 인식용)

### 2단계 · 설치 확인

새 PowerShell 창에서:

```powershell
node --version
npm --version
```

`v20.xxx.x` 같은 숫자가 두 번 나오면 OK. 만약 `'node'은(는) 내부 또는 외부 명령... 인식되지 않습니다` 가 나오면 → PowerShell을 닫고 한 번 더 새로 열어보세요. 그래도 안 되면 [트러블슈팅 FAQ](/extras/troubleshooting#node-not-found).

### 3단계 · 실행 정책 설정 (한 번만)

PowerShell은 기본적으로 일부 스크립트를 막아둡니다. **PowerShell을 관리자 권한으로** 새로 열고:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

`Y` 입력하고 Enter.

> 이게 뭐예요? "내 계정에서는 검증된 스크립트는 돌아가게 해줘"라는 설정입니다. npm 같은 도구가 정상 동작하려면 필요해요.

### 4단계 · Claude Code 설치

여전히 **관리자 권한 PowerShell**에서:

```powershell
npm install -g @anthropic-ai/claude-code
```

설치 메시지가 한참 흐른 뒤 끝나면:

```powershell
claude --version
```

버전 숫자가 나오면 성공. 안 나오면 [트러블슈팅 FAQ](/extras/troubleshooting#claude-not-found).

### 5단계 · 첫 실행

이제 **일반(관리자 아닌) PowerShell**에서:

```powershell
mkdir my-claude-test
cd my-claude-test
claude
```

→ [04 설치와 첫 실행](/start/install#3단계--첫-실행과-로그인) 의 로그인 단계로 이어집니다.

## 4. 옵션 B · Git Bash로 시작하기 (★ 비개발자 추천)

### 1단계 · Git for Windows 설치

1. 브라우저로 [git-scm.com/download/win](https://git-scm.com/download/win) 접속
2. 자동으로 다운로드 시작 (안 되면 페이지의 "Click here to download" 클릭)
3. 다운로드한 `.exe` 파일 더블클릭

### 2단계 · 설치 옵션 (이대로만 하세요)

설치 마법사가 길어요. 대부분 **"Next"** 누르면 되는데, 아래만 신경 써주세요.

| 화면 | 권장 선택 |
|---|---|
| Select Components | "Git Bash Here" 체크 (기본 체크) |
| Default editor | "Use Visual Studio Code as Git's default editor" (VS Code 깔려 있으면) / 아니면 그냥 Notepad |
| PATH environment | **"Git from the command line and 3rd-party software"** (가운데 옵션) |
| Line ending conversions | **"Checkout as-is, commit Unix-style"** |
| Terminal emulator | **"Use MinTTY"** (기본) |
| 나머지 | 모두 기본값 그대로 Next |

### 3단계 · Git Bash 열기

`Windows 키` → `git bash` → Enter.

처음 보는 화면이 떠요. 환영합니다 — 이게 터미널이에요.

```bash
echo "안녕"
```

이렇게 치고 Enter 해서 `안녕` 이 나오면 OK.

### 4단계 · Node.js 설치

Git Bash는 Node.js를 자동으로 깔아주지 않습니다. 별도 설치 필요해요.

1. [nodejs.org](https://nodejs.org) 접속 → **LTS** 다운로드 → 설치 (Add to PATH 체크 유지)
2. 설치 끝나면 **Git Bash를 닫고 새로 열기**
3. 확인:

```bash
node --version
npm --version
```

### 5단계 · npm 글로벌 폴더 권한 설정 (Git Bash 필수)

Git Bash에서 `npm install -g` 를 그냥 하면 권한 에러가 자주 납니다. 미리 막아두는 게 정신건강에 좋아요.

```bash
mkdir -p ~/npm-global
npm config set prefix ~/npm-global
echo 'export PATH=~/npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

> 이게 뭐예요? npm이 패키지를 깔 때 시스템 폴더 대신 **내 사용자 폴더**(`~/npm-global`)를 쓰도록 바꾸는 작업입니다. 권한 문제 거의 사라져요.

### 6단계 · Claude Code 설치

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

버전 숫자 나오면 성공.

### 7단계 · 첫 실행

```bash
mkdir my-claude-test
cd my-claude-test
claude
```

→ [04 설치와 첫 실행](/start/install#3단계--첫-실행과-로그인) 의 로그인 단계.

## 5. 옵션 C · WSL(Ubuntu)로 시작하기 (★ 개발자 추천)

### 1단계 · WSL 활성화

**PowerShell을 관리자 권한**으로 열고 ([2번 섹션](#powershell-여는-법) 참고):

```powershell
wsl --install
```

이 한 줄이 끝납니다. Windows 기능 활성화 + Ubuntu 자동 설치까지 해줘요.

### 2단계 · 재부팅

설치 끝났다는 메시지가 나오면 **컴퓨터 재부팅**. (필수입니다.)

### 3단계 · Ubuntu 첫 실행

재부팅 후, `Windows 키` → `ubuntu` → Enter.

검은 창이 뜨면서:
- "Enter new UNIX username:" → 짧은 영문 이름 입력 (예: `daniel`). 한글 X.
- "New password:" → 비밀번호 입력 (입력해도 화면에 아무 글자도 안 보임 — 정상!)
- "Retype new password:" → 같은 비밀번호 한 번 더

이 비밀번호는 나중에 `sudo` (관리자 권한 명령) 쓸 때 묻습니다. **꼭 기억하세요.**

### 4단계 · 패키지 업데이트

```bash
sudo apt update && sudo apt upgrade -y
```

비밀번호 한 번 묻고, 한참 줄줄 흐릅니다. 끝날 때까지 기다리세요.

### 5단계 · Node.js 설치

```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

확인:

```bash
node --version
npm --version
```

### 6단계 · Claude Code 설치

```bash
sudo npm install -g @anthropic-ai/claude-code
claude --version
```

### 7단계 · 작업 폴더 → Claude 실행

WSL에서는 **WSL 안의 폴더에서 작업하는 게 빠릅니다.** Windows 폴더(`/mnt/c/...`)는 가능은 하지만 느려요.

```bash
cd ~              # 내 홈 폴더로 (가장 빠른 곳)
mkdir my-claude-test
cd my-claude-test
claude
```

> 💡 **VS Code와 같이 쓰기**: VS Code 확장 "Remote - WSL" 깔면 VS Code에서 WSL 폴더를 직접 열 수 있어요. 가장 편합니다.

### Windows 파일에 접근하고 싶을 때

```bash
cd /mnt/c/Users/내이름/Desktop
```

`C:\Users\내이름\Desktop` 이 `/mnt/c/Users/내이름/Desktop` 으로 바뀐 셈이에요.

## 6. 한국어 / 인코딩 문제

Windows의 가장 짜증나는 부분.

### 한글이 `????` 또는 깨진 글자로 표시될 때

**PowerShell · Git Bash 즉시 해결**:

```bash
chcp 65001
```

이건 "이 창에서는 UTF-8 (한글 가능) 쓸게" 라는 뜻. 단, **창 닫으면 풀려요.**

### 영구 적용 — PowerShell

`$PROFILE` 파일을 한 번 열어 추가하면 매번 자동으로 적용됩니다.

```powershell
notepad $PROFILE
```

(파일이 없다고 뜨면 "예" 눌러서 만들기) 메모장에 아래 내용을 붙여넣고 저장:

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001 > $null
```

PowerShell 닫고 새로 열기 → 한글 깨짐 사라짐.

### 영구 적용 — Git Bash

```bash
echo "alias claude='LANG=ko_KR.UTF-8 claude'" >> ~/.bashrc
source ~/.bashrc
```

### WSL — 거의 문제 없음

깨질 일이 거의 없지만 만약 깨지면:

```bash
sudo locale-gen ko_KR.UTF-8
sudo update-locale LANG=ko_KR.UTF-8
```

### 파일 저장할 때 한글 깨짐

에디터 인코딩을 **UTF-8 (BOM 없음)** 으로 통일.
- VS Code: 우하단 인코딩 표시 클릭 → "Save with Encoding" → UTF-8

### 폴더/파일명에 한글 → 명령어가 인식 못 함

가능하면 **영문**으로 폴더 만드세요. 어쩔 수 없으면 따옴표로 감싸기:

```bash
cd "프로젝트 폴더"
```

## 환경 변수 편집 (클릭 단위)

`PATH`나 `GITHUB_TOKEN` 같은 환경 변수를 편집해야 할 때가 있어요. 클릭 단위로 안내합니다.

### 방법 1 — GUI (가장 쉬움)

1. **`Windows 키`** → 검색창에 `환경 변수` 입력
2. 검색 결과 **"시스템 환경 변수 편집"** 클릭
3. 창 아래쪽 **"환경 변수(N)..."** 버튼 클릭
4. 두 개의 표가 보임:
   - **위쪽**: 사용자 환경 변수 (이 사용자만)
   - **아래쪽**: 시스템 환경 변수 (모든 사용자, 관리자 권한 필요)
5. **새 변수 추가**: "새로 만들기(N)..." → 변수 이름·값 입력
6. **PATH에 경로 추가**: 위쪽 표에서 `Path` 더블클릭 → "새로 만들기" → 경로 붙여넣기 → 확인
7. **터미널을 닫고 새로 열어야** 적용됨

### 방법 2 — PowerShell

```powershell
# 일회성 (이 세션에서만)
$env:GITHUB_TOKEN = "ghp_xxxxx"

# 영구 (사용자 변수)
[Environment]::SetEnvironmentVariable("GITHUB_TOKEN", "ghp_xxxxx", "User")
```

### 방법 3 — Git Bash / WSL

```bash
echo 'export GITHUB_TOKEN="ghp_xxxxx"' >> ~/.bashrc
source ~/.bashrc
```

## 7. Windows에서 자주 만나는 에러

### `'claude'은(는) 내부 또는 외부 명령... 인식되지 않습니다`

PATH에 npm 글로벌 폴더가 없어서 그래요.

**확인**:
```bash
npm config get prefix
```

이 경로 안에 `claude.cmd` 가 있어야 해요. 없다면 → 설치 다시.
경로는 있는데 인식 못 한다면 → **터미널 새로 열기** (대부분 해결).

### `EACCES: permission denied`

npm 글로벌 폴더 권한 부족.

**Git Bash 해결** (옵션 B의 5단계 그대로):
```bash
mkdir -p ~/npm-global
npm config set prefix ~/npm-global
echo 'export PATH=~/npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

**PowerShell 해결**: 관리자 권한으로 다시 실행.

### `EPERM: operation not permitted`

대부분 백신 / Windows Defender 가 npm 관련 폴더를 차단해서 발생.

**해결**:
- Windows 보안 → 바이러스 및 위협 방지 → 제외 추가 → 폴더 추가
- 추가할 폴더: `C:\Users\내이름\AppData\Roaming\npm` 또는 npm prefix 경로

### 줄바꿈 (CRLF vs LF) 이슈

Windows가 다른 OS와 줄바꿈 방식이 다른 데서 오는 짜증. Git Bash 사용 시 한 번에 해결:

```bash
git config --global core.autocrlf input
git config --global core.eol lf
```

### 긴 경로 에러 (`Path too long`)

Windows의 260자 제한. PowerShell **관리자 권한**으로:

```powershell
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
                 -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
```

재부팅하면 풀려요.

### `Cannot find module 'X'` (WSL과 Windows 혼용 시)

같은 폴더에서 WSL에서도 `npm install`, Windows에서도 `npm install` 했을 때 발생. **한 폴더는 한 환경에서만** install 하세요.

## 8. 추천 도구

### Windows Terminal (강력 추천)

기본 PowerShell 창은 못생기고 불편해요. **Windows Terminal**은 Microsoft가 만든 무료 터미널인데, 탭·분할·테마 다 됩니다.

설치: Microsoft Store에서 "Windows Terminal" 검색 → 설치.
설정 한 번 해두면 PowerShell·Git Bash·WSL **하나의 창에서 탭으로 다 다룰 수 있어요.**

### VS Code

가장 많이 쓰는 무료 에디터. Claude Code와 통합도 잘 됩니다 (`claude --ide`).

- 한글 폰트 추천: **D2Coding**, **Pretendard Mono**, **JetBrains Mono**
- WSL 쓰면 "Remote - WSL" 확장 필수

### Windows Terminal에서 한글 폰트 설정

Settings → 사용 중인 Profile → Appearance → Font face → `D2Coding` 또는 `Cascadia Code KR`
