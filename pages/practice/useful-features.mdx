# 14 · 알면 편한 기능 모음

> 안 알면 손해, 알면 인생 편해지는 기능들.

## 1. `!` 셸 명령 한 번에

`> !명령어` 로 입력하면 그 명령이 셸에서 실행되고 결과가 대화에 들어옵니다.

```
> !ls -la
> !git status
> !curl https://api.example.com/data | head -20
```

> 💡 사용자가 직접 친 명령이라 권한 묻지 않음. 빠릅니다.

## 2. `@` 파일/폴더 명시 첨부

```
> @README.md 이 문서 핵심만 정리해줘
> @src/components 이 폴더 구조 설명해줘
```

특정 파일/폴더를 정확히 지목할 때.

## 3. 여러 줄 입력

엔터 = 전송이 기본인데, 코드나 긴 글을 붙여넣을 땐:
- `Shift + Enter` 로 줄바꿈
- 또는 ` ``` ` 로 시작 → 마지막에 ` ``` ` 로 닫기

## 4. 입력 히스토리

`↑` / `↓` 키로 이전에 친 명령 다시 불러오기. 비슷한 부탁 반복할 때 편함.

## 5. 자동 완성 (Tab)

파일명 / 명령어 입력 중 `Tab` 키로 자동완성.

## 6. `/cost` 토큰 사용량 즉시 확인

```
/cost
```

이번 세션에서 얼마나 썼는지. 비싼 모델로 길게 작업할 땐 자주.

## 7. `/model` 모델 빠른 전환

복잡 작업 → Opus, 단순 정리 → Sonnet, 초간단 → Haiku.

```
/model
```

선택 메뉴가 나옵니다.

## 8. Plan 모드 (안전 미리보기)

뭔가 위험해보이는 작업 전에:

```
/permissions plan
```

이러면 Claude는 **아무것도 안 바꾸고 계획만** 보여줘요. 검토 후 OK면 default로 돌아가서 진행.

## 9. 백그라운드 명령

오래 걸리는 작업은 백그라운드로:

```
> npm run build 백그라운드로 돌리고, 끝나면 알려줘.
```

Claude는 다른 일을 하면서 빌드를 모니터링합니다.

## 10. `/compact` 로 대화 다이어트

길어진 대화를 핵심만 남기고 압축:

```
/compact "코드 변경만 자세히, 잡담은 줄여서"
```

## 11. 재시작 없는 settings 갱신

`.claude/settings.json` 을 수정하면 Claude가 자동으로 다시 읽어요. 일부 설정은 재실행 필요.

## 12. 터미널 멀티 세션

작업 폴더가 다르면 터미널 창 여러 개 띄워서 각각 `claude` 실행. 동시 진행.

## 13. `/init` 으로 CLAUDE.md 자동 생성

새 폴더에서 첫 명령으로 추천. 폴더 구조 분석해서 초안 만들어줍니다.

## 14. `/memory` 로 장기 기억 보기

```
/memory
```

지금까지 쌓인 내 정보, 피드백, 프로젝트 메모를 한눈에. 잘못된 게 있으면 즉시 수정.

## 15. 에러 자동 분석 패턴

코드 실행 후 에러가 나면, 결과를 그대로 두고:

```
> 위 에러 원인 분석해줘
```

Claude는 직전 명령의 출력을 보고 진단해요.

## 16. 숨은 슬래시 명령들

```
/help
```

으로 전체 목록. 사용 가능한 모든 명령이 나옵니다. 알려진 외에도:

- `/feedback` — Anthropic에 피드백
- `/upgrade` — Claude Code 자체 업데이트
- `/release-notes` — 최근 변경사항
- `/login` — 계정 전환

## 17. `claude --resume`

이전 세션 이어서 시작:

```bash
claude --resume
```

마지막 세션의 컨텍스트를 일부 가져옵니다 (환경에 따라 가능 여부 다름).

## 18. 권한을 영구 등록

권한 묻는 화면에서 **"Yes, and don't ask again for similar"** 누르면 자동 settings.json에 추가됩니다. 자주 쓰는 명령 등록 좋아요.

## 19. 출력을 파일로

```
> 이 분석 결과를 analysis.md 파일로 저장해줘
```

긴 결과를 다음에 다시 참고할 수 있게.

## 20. 다른 폴더 빠른 점프

```
> ../another-project/ 폴더 구조 한 번 봐줘
```

상대 경로로 다른 폴더 잠깐 들여다보기.

## 21. CLAUDE.md `@import`

```markdown
# 코드 컨벤션
@docs/coding-style.md 참조
```

다른 파일을 끌어오는 문법.

## 22. 색상 출력

```
> 결과를 ANSI 컬러 코드 써서 보여줘
```

지원되는 터미널이면 색상으로 강조.

## 23. 진행 상태 추적 (TaskCreate)

긴 작업은 Claude가 자체 task list로 진행 상황 추적해줘요. 화면에서 확인 가능.

## 24. 비동기 명령

```
> 빌드 돌리고 (백그라운드), 그동안 README 작성해줘
```

병렬 진행으로 시간 절약.

## 25. 한국어 입력 토글

CLAUDE.md에 "기본 언어는 한국어" 적어두면 매번 부탁할 때 한국어로 답해줘요. 영어 답이 필요하면 그때만 영어로 부탁.
