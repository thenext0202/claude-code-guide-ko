# 13 · 디버깅 요청하기

> 에러는 적이 아니에요. **정보**입니다. 잘 전달하면 5분 안에 해결됩니다.

## 디버깅 좋은 부탁 vs 나쁜 부탁

### ❌ 나쁜 예
```
> 안 돼요. 고쳐주세요.
```
정보가 없음. Claude가 추측해야 함.

### ✅ 좋은 예
```
> 다음 명령 실행했더니 에러가 나. 원인 찾아서 고쳐줘.

   명령: npm run dev
   에러: Module not found: Can't resolve 'next/router' in '/Users/me/blog/pages/index.tsx'
   환경: Node v20, Next.js 14
   언제부터: 어제까지 됐는데, 오늘 npm install 한 후부터.
```

## 디버깅 만능 5요소

좋은 부탁에 들어가야 할 것들.

### 1. **명령** — 무엇을 했는데?
```
npm run dev
git push origin main
python script.py --input data.csv
```

### 2. **에러 메시지 전체** — 일부만 X, 전체 O
```
ModuleNotFoundError: No module named 'pandas'
  File "/Users/me/script.py", line 3, in <module>
    import pandas as pd
```

> 💡 에러는 마지막 줄만 보지 말고 **stack trace 전체**가 더 가치 있어요.

### 3. **환경** — OS, 버전
```
- macOS 14.5
- Node v20.10.0
- Python 3.11.5
- 패키지 버전: pandas 2.2.0
```

### 4. **언제부터** — 변경 시점
```
"어제까지는 됐어. 오늘 아침 git pull 한 직후부터."
"npm install --save react-router 한 다음부터."
"파일 하나 추가한 후부터."
```

### 5. **재현 방법** — 어떻게 다시 일어나나
```
1. 홈페이지 접속
2. 로그인 버튼 클릭
3. 이메일/비밀번호 입력
4. 제출 버튼 → 여기서 화면 멈춤
```

## 단계별 디버깅 부탁 패턴

### 1단계 · 진단부터
```
> 위 에러의 원인을 분석해줘. **고치진 말고**, 무엇이 문제인지 추정 3가지를 알려줘.
```

### 2단계 · 우선순위 결정
```
> 그중 가장 가능성 높은 1번부터 확인해보자.
   먼저 [추정 원인 1] 이 맞는지 어떻게 검증할 수 있어?
```

### 3단계 · 검증
```
> 알겠어. 그 명령 실행해줘.
```

### 4단계 · 수정
```
> 결과로 보니 [원인]이 맞네. 이 부분 고쳐줘.
   단, 변경 전후 비교를 git diff로 보여줘.
```

이렇게 하면 **AI가 무작정 고치지 않고 같이 추론**합니다.

## 자주 만나는 에러별 빠른 가이드

### `command not found: claude`
- 설치 안 됐거나 PATH에 없음
- → `npm install -g @anthropic-ai/claude-code`
- → 그래도 안 되면 [트러블슈팅](/extras/troubleshooting#claude-not-found)

### `Module not found: ...`
- 패키지 미설치 또는 잘못된 경로
- → `npm install [패키지명]`
- → 또는 import 경로 오타 확인

### `Permission denied`
- 파일 권한 부족
- macOS/Linux: `chmod +x [파일]` 또는 `sudo`
- → 비밀번호 묻는 명령은 `!` 프리픽스로 직접 실행

### `EADDRINUSE: address already in use :::3000`
- 3000번 포트가 다른 프로세스에 잡혀있음
- → `lsof -i :3000` 으로 찾아서 `kill <PID>`
- → 또는 다른 포트 사용

### `git pull` 에서 conflict
```
> git pull 했더니 conflict 났어. 자동 머지 말고
   수동으로 해결할 부분 알려주고, 어떻게 결정할지 같이 정해줘.
```

### Python `IndentationError`
- 들여쓰기 섞임 (탭 vs 스페이스)
- → "이 파일 들여쓰기 통일해줘. 4칸 스페이스로."

### Hot reload 안 됨
- 파일 시스템 watcher 한도 (macOS) 또는 캐시 문제
- → 서버 재시작
- → `node_modules/.cache` 삭제

## 디버깅 시 절대 하지 말아야 할 것

### ❌ 무작정 코드 다 갈아엎기
```
> 일단 다 다시 짜줘
```
원인 파악 없이 새로 짜면 같은 문제 재발 가능. **진단 먼저.**

### ❌ 에러 무시하고 다음 단계
```
> 에러 나는데 일단 빌드는 됐으니 배포해줘
```
배포 후 터집니다.

### ❌ 깊은 검증 없이 "고쳐졌어요" 믿기
- 직접 테스트 케이스 돌려보기
- `git diff` 로 변경 검토
- 실제 사용 흐름 재현

## 어려운 에러일 땐

### 로그 모아서 줘봐
```
> debug.log 파일에 모든 로그 다 적어줘.
   그 파일 읽고 원인 분석해.
```

### 비슷한 사례 검색
```
> 이 에러를 영어로 번역해서 GitHub Issues / StackOverflow에서 검색해줘.
   유사 사례 3개 요약하고, 어떤 해결책이 있었는지 알려줘.
```

### 최소 재현 코드 만들기
```
> 이 문제를 재현하는 가장 작은 코드 (10줄 이내) 만들어줘.
   그걸로 다시 디버깅하자.
```
