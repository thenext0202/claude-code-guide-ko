# 04 · 설치와 첫 실행

> 이 페이지는 **"터미널이라는 게 뭔지도 모르겠다"** 는 분 기준으로 썼어요. 천천히 한 줄씩 따라오면 15~20분 안에 끝납니다.

## 시작 전에 — 터미널이 뭔지부터

평소엔 마우스로 폴더를 더블클릭해서 파일을 열죠. **터미널은 그걸 글자로 하는 도구**예요.

검은(또는 파란) 화면에 깜빡이는 커서가 있고, 거기에 명령어 한 줄을 치면 컴퓨터가 그대로 실행해줍니다. **Claude Code는 이 터미널 안에서 동작해요.**

처음엔 어색한데 이번 가이드로 5~6개의 명령어만 익히면 충분합니다.

> 📚 **이 페이지를 보기 전에 알면 좋은 것** (몰라도 따라올 수 있지만, 깊이 이해하려면)
> - [📖 용어집](/extras/glossary) — 터미널·셸·CLI·PATH 같은 단어 사전
> - [⌨️ 터미널 기초 명령어](/extras/terminal-basics) — `cd`, `ls`, `pwd` 등이 뭔지
> - [✏️ 에디터 셋업](/extras/editor-setup) — 설정 파일을 만질 수 있는 환경 만들기

> 🪟 **Windows 사용자라면 잠깐!** Windows에는 터미널 종류가 3가지(PowerShell / Git Bash / WSL)인데 어느 걸 고르느냐가 중요해요. **[Windows 환경 풀가이드](/extras/windows-setup) 의 1번 섹션을 먼저 보고 오세요.** (5분이면 됩니다.) 거기서 터미널을 골라 설치까지 마쳤다면 이 페이지는 건너뛰셔도 돼요.
>
> 🍎 **macOS 사용자**: 별도 결정 없이 그냥 따라오시면 돼요. 기본 "터미널" 앱을 씁니다.

---

## 큰 그림

설치는 3단계입니다.

1. **터미널 열기** — 명령어 칠 무대 준비
2. **Node.js 설치** — Claude Code가 돌아가는 기반 도구
3. **Claude Code 설치 + 로그인** — 본 게임

각 단계 끝마다 **"제대로 됐나 확인하는 명령어"** 가 있어요. 그게 통과해야 다음 단계로 넘어가세요. **건너뛰면 나중에 더 헤맵니다.**

---

## 1단계 · 터미널 열기

### 🍎 macOS

1. `Command(⌘) + Space` → 스포트라이트 검색창
2. **`터미널`** 또는 **`terminal`** 타이핑 → Enter
3. 창이 뜨면 OK

### 🪟 Windows

[Windows 풀가이드](/extras/windows-setup#2-터미널-여는-법-클릭-단위-안내) 에서 자세히 다룹니다. 빠른 요약:

- **PowerShell**: `Windows 키` → `powershell` 입력 → Enter
- **Git Bash**: `Windows 키` → `git bash` 입력 → Enter (Git 설치돼 있어야 함)
- **WSL/Ubuntu**: `Windows 키` → `ubuntu` 입력 → Enter (WSL 설치돼 있어야 함)

### 🐧 Linux

`Ctrl + Alt + T` 또는 앱 메뉴에서 "Terminal".

### 첫 시험

터미널이 떴으면 한 번 쳐보세요:

```bash
echo "안녕"
```

`Enter` 누르면 다음 줄에 `안녕` 이 나옵니다. **이걸로 "터미널이 내 명령을 받는다"는 게 확인됐어요.** 첫 관문 통과.

> 💡 **명령어 복사 팁**: 이 가이드의 코드 블록은 우상단에 복사 버튼이 있어요. 그걸 누르고 터미널에 붙여넣기:
> - macOS: `Cmd + V`
> - Windows: `Ctrl + V` (또는 우클릭 한 번)
> - Linux: `Ctrl + Shift + V`

## 2단계 · Node.js 설치

> Node.js는 자바스크립트를 컴퓨터에서 돌리는 엔진. Claude Code가 이 위에서 동작합니다. 한 번만 깔면 돼요.

### 🍎 macOS

**가장 쉬움**:
1. 브라우저로 [nodejs.org](https://nodejs.org) 접속
2. 큰 초록 버튼 **"LTS"** 클릭 → `.pkg` 파일 다운로드
3. 다운로드한 파일 더블클릭 → **계속, 계속, 동의, 설치**
4. 비밀번호 한 번 묻습니다 (Mac 로그인 비번)
5. 끝나면 **터미널을 닫고 새로 열기**

### 🪟 Windows

> 이미 Windows 풀가이드를 따라 환경 설정을 마쳤다면 거기서 Node.js까지 같이 깔았을 거예요. 그러면 이 단계는 건너뛰고 **3단계로 가도 됩니다.**

아직 안 깔았다면:
1. 브라우저로 [nodejs.org](https://nodejs.org) 접속
2. **"LTS"** 버튼 → `.msi` 다운로드
3. 다운로드 파일 더블클릭 → Next → **"Add to PATH" 체크 확인** → Next → Install
4. 끝나면 **터미널 창을 닫고 새로 열기** (중요)

### 🐧 Linux (Ubuntu/Debian)

```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 설치 확인 (필수)

새 터미널에서:

```bash
node --version
npm --version
```

각 줄에 `v20.xxx.x` / `10.x.x` 같은 숫자가 나오면 OK.

만약 `command not found` 또는 `'node'은(는) 내부 또는 외부 명령... 인식되지 않습니다`:
- 터미널을 한 번 더 닫고 새로 열어보세요 (PATH 인식 문제일 수 있음)
- 그래도 안 되면 → [트러블슈팅 → node not found](/extras/troubleshooting#node-not-found)

## 3단계 · Claude Code 설치

명령어 한 줄입니다.

```bash
npm install -g @anthropic-ai/claude-code
```

> **Windows PowerShell** 사용자: 이 명령은 **관리자 권한 PowerShell**에서 실행하세요. (`Windows 키` → `powershell` → 우클릭 → "관리자 권한으로 실행")
>
> **macOS / Linux** 사용자: 권한 에러(`EACCES`) 가 나면 `sudo` 를 앞에 붙이지 말고 → [npm 권한 설정](/extras/troubleshooting#npm-permission) 의 방식으로 한 번 정리하는 걸 권장합니다.

### 무슨 일이 일어났나요?
- `npm`: Node.js의 패키지 매니저 (앱스토어 같은 것)
- `install -g`: 컴퓨터 전체에 설치 (`g` = global)
- `@anthropic-ai/claude-code`: 패키지 이름

설치는 30초~2분 정도 걸립니다. 메시지가 줄줄 흘러가다 멈추면 끝.

### 설치 확인 (필수)

```bash
claude --version
```

버전 숫자가 나오면 성공.

#### 안 나오면?

| 증상 | 어디로 |
|---|---|
| `command not found` / `인식되지 않습니다` | [트러블슈팅 → claude not found](/extras/troubleshooting#claude-not-found) |
| `EACCES: permission denied` | [트러블슈팅 → npm 권한](/extras/troubleshooting#npm-permission) |
| Windows에서 한글 깨짐 | [Windows 가이드 → 인코딩](/extras/windows-setup#6-한국어--인코딩-문제) |

## 4단계 · 첫 실행과 로그인

### 4-1. 작업 폴더 만들기

새 폴더 하나 만들어서 거기서 시작하는 게 좋아요. **Claude Code는 "현재 위치한 폴더" 안의 파일들을 봅니다.** 그러니 빈 연습용 폴더를 먼저 만드세요.

```bash
mkdir my-claude-test
cd my-claude-test
```

> 명령어 풀이:
> - `mkdir my-claude-test` → "my-claude-test 라는 폴더를 만들어"
> - `cd my-claude-test` → "그 폴더 안으로 들어가"
>
> 지금 어디 있는지 확인은:
> - macOS/Linux/WSL/Git Bash: `pwd`
> - Windows PowerShell: `pwd` 또는 `Get-Location`
>
> 다른 명령어가 더 궁금하면 → [⌨️ 터미널 기초](/extras/terminal-basics)

### 4-2. Claude 실행

```bash
claude
```

이런 환영 화면이 나옵니다:

```
╭───────────────────────────╮
│  ✻ Welcome to Claude Code  │
╰───────────────────────────╯

Sign in to your Anthropic account...
```

### 4-3. 로그인

`Enter` 를 누르면 **자동으로 브라우저가 열려요.**

1. Anthropic 계정으로 로그인 (없으면 [claude.ai](https://claude.ai) 가입)
2. 브라우저에서 권한 허용 클릭
3. 자동으로 터미널로 돌아옴

> ❌ **브라우저가 안 열리거나 흐름이 막힐 때**: 화면에 표시된 **URL을 직접 복사** → 브라우저에 붙여넣기 → 그래도 안 되면 [로그인 트러블슈팅](/extras/troubleshooting#login).

## 5단계 · 첫 대화 (드디어!)

로그인되면 입력창이 나옵니다 (`>` 모양). 한국어로 부탁해보세요.

```
> 이 폴더에 뭐가 있는지 알려줘
```

비어있는 폴더라 "비어있어요" 같은 답이 옵니다. 다음:

```
> hello.txt 파일 만들어서 "안녕 클로드 코드" 라고 적어줘
```

**Yes / No** 같은 권한 확인이 뜰 거예요. 화살표로 **Yes** 선택 → Enter. 파일이 생성됩니다.

### 만들어진 파일 확인

다른 터미널 창을 열거나 Claude를 잠시 종료(`/exit`)하고:

```bash
ls          # 폴더 안 파일 목록
cat hello.txt   # macOS / Linux / Git Bash / WSL
type hello.txt  # Windows PowerShell
```

`안녕 클로드 코드` 가 보이면:

🎉 **설치 완료. AI와의 첫 협업.**

---

## Claude 종료하는 법

세 가지 다 OK:
- 입력창에 `/exit` 입력 + Enter
- `Ctrl + C` 두 번 빠르게
- 터미널 창을 그냥 닫기

다시 시작하고 싶으면 똑같이 그 폴더에서 `claude` 만 치면 돼요.

## 자주 나오는 첫날 막힘

| 증상 | 원인·해결 |
|---|---|
| `claude: command not found` (또는 인식되지 않습니다) | 설치가 끝났는지 확인 / 터미널 새로 열기 → [상세](/extras/troubleshooting#claude-not-found) |
| `EACCES / permission denied` | npm 글로벌 폴더 권한 → [상세](/extras/troubleshooting#npm-permission) |
| 로그인 페이지가 끝나지 않음 | URL 직접 복사 / 회사 네트워크 의심 → [상세](/extras/troubleshooting#login) |
| Windows에서 한글 `????` | `chcp 65001` 또는 [영구 적용](/extras/windows-setup#6-한국어--인코딩-문제) |
| Yes/No 같은 게 자꾸 떠서 답답 | 정상이에요. Claude가 안전을 위해 묻는 거. 자주 쓰는 작업은 나중에 [트러블슈팅 FAQ](/extras/troubleshooting) 의 "권한 묻는 게 매번 떠서 답답" 항목으로 자동 허용 가능 |
