# 05 · 초보자 워크플로우

> "켰는데… 뭘 어떻게 시키지?"

이 페이지는 **모든 작업에 통하는 5단계 패턴**입니다. 일단 이대로 따라 하면 됩니다.

> 📚 **이 페이지 끝나면 할 수 있는 것**
> - 어떤 작업이든 통하는 **5단계 패턴** 익히기
> - 첫 5일치 **연습 코스** 따라하기 (한 줄씩 스크립트 제공)
> - "잘 시키는 7가지" 와 "자주 하는 5가지 실수" 미리 알고 가기
>
> 🔗 함께 보면 좋은 페이지: [04 설치](/start/install) · [09 자주 쓰는 명령어](/core/commands) · [12 자주 하는 실수](/practice/common-mistakes)

## 만능 5단계 패턴

```
1. 폴더 준비       ← 작업할 파일이 있는 곳으로 이동
2. claude 실행
3. 의도 설명       ← "내가 하고 싶은 건 ~ 야"
4. 단계별 부탁     ← 한 번에 다 시키지 않고 한 단계씩
5. 결과 확인 / 반복  ← Yes/No 잘 보고, 결과 검토
```

이 흐름만 잡으면, 어떤 작업이든 해낼 수 있어요.

## 실전 예시: 회의록 30개 요약

### 1단계 · 폴더 준비

회의록 파일들을 한 폴더에 모아둡니다.
```
~/Desktop/meeting-notes/
├── 2026-01-15.md
├── 2026-01-22.md
├── 2026-01-29.md
... (30개)
```

### 2단계 · 그 폴더에서 claude 실행

```bash
cd ~/Desktop/meeting-notes
claude
```

### 3단계 · 의도 설명 (가장 중요)

이게 핵심이에요. **무엇을, 어떻게, 왜** 하고 싶은지 한 번에 알려주세요.

```
> 이 폴더에 1월부터 3월까지 매주 회의록이 있어.
   각 회의록을 5줄로 요약해서 summary.md 한 파일에
   날짜순으로 정리해줘. 결정사항이 있으면 ✅ 표시 붙여서.
```

> 💡 **이 한 문장에 들어간 것들**: 폴더 위치, 입력(회의록), 출력(summary.md), 형식(5줄, 날짜순), 특별 규칙(결정사항 ✅)

### 4단계 · 단계별 진행

Claude는 보통 큰 작업을 작은 단계로 쪼갭니다.

```
✻ 30개 파일 확인하고 있습니다... (Read tool)
✻ 각 파일을 5줄 요약하고 있습니다...
✻ summary.md 파일을 만들고 있습니다... (Write tool)

→ 진행 OK?
```

여기서 **Yes** 누르면 진행, **No** 누르면 멈춤.

### 5단계 · 결과 확인

```bash
cat summary.md
```

으로 결과를 직접 봅니다. 마음에 안 들면 추가 부탁:

```
> ✅ 표시가 빠진 것 같아. 다시 한 번 확인해줘.
   그리고 매주 참석자 명단도 추가해줘.
```

이게 **반복**입니다. AI에게 한 번에 완벽한 결과를 받으려 하지 마세요. **대화하면서 다듬는 것**이 정답.

## 처음 며칠 추천 연습 코스

> 5일치 워크숍입니다. 하루에 다 하지 말고, 하루에 하나씩. 매일 30분 정도면 충분.

### 📅 Day 1 · 첫 대화 워밍업 (15분)

> 목표: Claude Code와 **세 번** 대화해보기. 파일 만들기·읽기·수정 한 번씩.

#### Step 1. 빈 연습 폴더 만들기 (터미널에서)

```bash
cd ~                              # 홈 폴더로
mkdir claude-day1                 # 연습용 폴더 만들기
cd claude-day1                    # 그 안으로 들어가기
pwd                               # 지금 어디 있는지 확인
```

`pwd` 결과가 `.../claude-day1` 같이 끝나면 OK.

#### Step 2. Claude 실행

```bash
claude
```

환영 화면이 뜹니다. 이제 `>` 모양 입력창이 나오면 시작할 준비 완료.

#### Step 3. 첫 질문 — 폴더 보기

```
> 이 폴더에 뭐가 있어?
```

Claude가 `Read tool` / `LS tool` 같은 표시와 함께 "이 폴더는 비어있어요" 같은 답을 줍니다.

> 💡 화면에 `(LS tool)` 같은 게 잠깐 뜨는 게 정상이에요. Claude가 "지금 LS 도구를 쓰고 있다"는 신호.

#### Step 4. 두 번째 — 파일 만들기

```
> hello.txt 파일 만들어서 "안녕 클로드" 라고 적어줘
```

이번엔 **Yes/No 권한 확인**이 뜰 거예요:

```
🔒 Tool: Write
   File: hello.txt
   Content: "안녕 클로드"

   1. Yes
   2. Yes, and don't ask again for similar
   3. No, and tell Claude what to do differently
```

화살표 키로 **1번 Yes** 선택 → Enter. 파일이 만들어집니다.

#### Step 5. 세 번째 — 파일 번역해서 새 파일 만들기

```
> hello.txt 내용을 영어로 번역해서 hello_en.txt 로 저장해줘
```

Read → Write 두 단계가 진행될 거예요. 각 단계마다 권한 확인 → Yes.

#### Step 6. 결과 확인

`/exit` 로 Claude 종료한 다음:

```bash
ls                       # 두 파일이 보여야 함
cat hello.txt            # "안녕 클로드"
cat hello_en.txt         # "Hello Claude" 같은 영문
```

> 🎉 첫날 끝. 이게 Claude Code의 가장 기본 흐름이에요. **부탁 → 권한 → 결과 확인.**

### 📅 Day 2 · 실전 작업 — 폴더 정리 (30분)

> 목표: 실제로 어지러운 폴더를 정리시켜보기. **계획 → 검토 → 실행** 패턴 익히기.

#### Step 1. 정리할 폴더로 이동

다운로드 폴더가 무난해요 (실제 망쳐도 괜찮은 파일들이라).

```bash
cd ~/Downloads     # Mac/Linux/WSL/Git Bash
# 또는
cd $env:USERPROFILE\Downloads     # Windows PowerShell
```

> ⚠️ **꼭 백업부터.** 중요한 파일이 있으면 다른 폴더로 복사하거나, [git 시작 패턴](/practice/github)으로 백업.

#### Step 2. Claude 실행 + 계획만 받기

```bash
claude
```

```
> 이 폴더 너무 지저분해. 파일 종류별로 정리하고 싶어.

   단, 다음 두 가지만 지켜줘:
   1. 어떻게 정리할지 **계획만 먼저** 보여줘. 실행은 내가 OK 하면.
   2. 원본 파일 절대 삭제하지 말고, 옮기는 것까지만.
```

Claude가 다음 같은 답을 줄 거예요:
```
계획:
- images/ 폴더 만들기 → .jpg, .png, .gif 이동
- pdfs/ 폴더 만들기 → .pdf 이동
- 기타/ 폴더 만들기 → 분류 안 되는 것
- 30일 이상 된 파일은 archive/ 로

진행할까요?
```

#### Step 3. 검토 → 진행

마음에 들면:

```
> 좋아. 그대로 진행해줘. 단, 각 파일 옮기기 전에 한 번씩 더 확인은 안 해도 돼.
```

또는 수정 요청:

```
> images/ 말고 사진/, pdfs/ 말고 문서/ 처럼 한국어로 폴더명 지어줘.
```

#### Step 4. 결과 확인

```
> 정리 끝났어? 결과 요약해줘. 어떤 파일이 어디로 갔는지.
```

Claude가 표나 목록으로 보고. 마음에 안 드는 게 있으면 추가 부탁.

> 💡 **Day 2의 교훈**: "계획만 먼저" 라는 한 마디로 위험을 크게 줄일 수 있어요. 큰 작업일수록 이 패턴 필수.

### 📅 Day 3 · CLAUDE.md 만들기 (20분)

> 목표: 매번 같은 설명을 반복하지 않게 **장기 기억** 만들기.

#### Step 1. 작업 폴더로 이동

본인이 자주 쓰는 폴더 하나 정해서 (마케팅 자료, 사이드 프로젝트, 회의록 등):

```bash
cd ~/Desktop/내폴더
claude
```

#### Step 2. `/init` 으로 자동 초안

```
/init
```

Claude가 폴더 안 파일들을 분석해서 CLAUDE.md 초안을 자동 생성해줍니다.

#### Step 3. 본인 입맛으로 다듬기

다른 터미널 창 또는 VS Code로 `CLAUDE.md` 열고 편집. (에디터 셋업이 안 되어 있으면 [✏️ 에디터 셋업](/extras/editor-setup) 먼저)

또는 Claude에게 부탁:

```
> CLAUDE.md에 다음 내용 추가해줘:

   # 이 폴더의 규칙
   - 파일명은 한국어로 짓는다
   - 회의록은 meeting_YYYY-MM-DD.md 형식
   - 결정 사항은 ✅, 미정은 ❓, 액션은 ⚡로 표시

   # 절대 금지
   - 원본 파일 덮어쓰기
   - 파일 삭제 (이동만)
```

#### Step 4. 효과 검증

`/clear` 로 대화 비우고:

```
> 회의록 정리해줘
```

Claude가 **CLAUDE.md를 자동으로 읽고 ✅/❓/⚡ 규칙을 따르는지** 확인. 따르면 성공.

→ 자세한 작성법: [08 CLAUDE.md](/core/claude-md)

### 📅 Day 4 · GitHub 연결 (30분, 코드 작업하는 분만)

> 목표: 작업한 폴더를 GitHub에 안전망으로 올려두기.

#### Step 1. 사전 준비 (한 번만)

- [github.com](https://github.com) 가입
- GitHub CLI 설치:
  ```bash
  brew install gh           # Mac
  winget install GitHub.cli # Windows
  sudo apt install gh       # Linux
  ```
- 로그인:
  ```bash
  gh auth login
  ```
  → 안내 따라 브라우저 인증

#### Step 2. Claude에게 통째로 부탁

```bash
cd ~/Desktop/내프로젝트
claude
```

```
> 이 폴더를 GitHub에 새 레포로 올려줘.
   - 레포 이름: my-first-claude-project
   - 비공개(Private)로
   - .gitignore 적절히 만들어줘 (.env, node_modules 등 제외)
   - 첫 커밋 메시지: "initial commit"
```

각 단계 권한 확인 잘 보고 Yes.

#### Step 3. 결과 확인

```
> github.com에서 레포 잘 만들어졌는지 확인해줘. 링크도 알려줘.
```

→ 자세히는 [11 GitHub와 함께 쓰기](/practice/github)

## 잘 시키는 7가지 팁

### 1. **목적**을 먼저 말해라
❌ "엑셀 합쳐줘"  
✅ "월별 매출 보고서 만들려고 해. 12개 엑셀을 합쳐서 한 시트로 정리해줘"

### 2. **출력 형식**을 명확히
❌ "정리해줘"  
✅ "Markdown 표 형식으로, 헤더는 한국어, summary.md 파일로 저장"

### 3. **예시**가 가장 강력
```
> 이런 식으로 정리해줘:
| 날짜 | 매출 | 결정사항 |
|---|---|---|
| 1/15 | 100만 | ✅ 광고 예산 증액 |
```

### 4. **금지사항**도 포함
> 단, 원본 파일은 삭제하지 말고 백업 폴더로 옮겨줘.

### 5. **확인 단계** 요청
> 먼저 어떻게 처리할지 계획만 보여주고, 내가 OK 하면 시작해.

### 6. **모르면 물어봐** 라고 말해주기
> 애매한 부분은 추측하지 말고 나한테 물어봐.

### 7. **권한 확인 잘 보기**
파일을 지우거나 큰 변경을 할 때 Yes/No 확인이 떠요. **그냥 Yes 누르지 말고**, 무엇을 한다는지 한 번 읽으세요.

## 처음 며칠 자주 하는 실수

| 실수 | 어떻게 고치나 |
|---|---|
| 한 번에 너무 큰 일 시킴 | 작게 쪼개기 |
| 의도 설명 없이 명령만 함 | "왜 하는지" 도 알려주기 |
| 결과 확인 안 함 | 항상 결과물 직접 열어보기 |
| 권한 묻는 거 무시하고 Yes | 무엇을 한다는지 읽기 |
| `/clear` 안 써서 컨텍스트 누적 | 새 작업 땐 `/clear` |

자세한 건 [12 자주 하는 실수](/practice/common-mistakes)에서.
