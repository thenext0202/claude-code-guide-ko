# 17 · Skills — 반복 작업 레시피

## Skills가 뭐예요?

> **Skills = 자주 하는 작업을 저장해두고 한 줄로 호출하는 레시피**

매번 "이 폴더에서 회의록 30개 정리하는데, 첫 줄에 참석자 명단, 결정은 ✅..." 라고 길게 부탁하지 말고, **`/회의록정리`** 한 줄로 끝.

## Plugin/MCP/Hooks와 차이?

| 기능 | 누가 부르나? | 어디에 산다? |
|---|---|---|
| **Plugin** | UI/명령 확장 | 플러그인 마켓 |
| **MCP** | 외부 서비스 연결 | 별도 프로세스 |
| **Hooks** | Claude가 자동 호출 | 도구 사용 시점에 |
| **Skills** | **사용자가 명시 호출** | `~/.claude/skills/` |

Skills는 **본인이 직접 만들기 가장 쉬움.**

## 만드는 법

### 1. 폴더 구조

```
~/.claude/skills/
└── meeting-notes/
    └── SKILL.md
```

스킬 하나당 폴더 하나. 그 안에 `SKILL.md` 파일.

### 2. SKILL.md 작성

```markdown
---
name: meeting-notes
description: 회의록 폴더에서 모든 회의록을 한 페이지로 요약합니다.
---

# 회의록 정리

이 스킬은 다음 작업을 합니다:

1. 현재 폴더에서 `*.md` 파일을 모두 찾는다.
2. 각 파일을 5줄로 요약한다.
3. `summary.md` 라는 파일로 날짜순 정리한다.
4. 결정 사항이 있으면 ✅ 표시.
5. 미정 사항은 ❓.
6. 액션 아이템은 ⚡ + 담당자.

**마지막으로 결과 파일 위치를 알려주세요.**
```

### 3. 실행

Claude를 켠 다음:

```
/meeting-notes
```

끝. 위에 적은 작업이 자동 진행됩니다.

## 매개변수 받기

```markdown
---
name: translate
description: 폴더의 마크다운 파일을 지정한 언어로 번역.
---

# 번역

사용자가 다음 인자를 넘겨줍니다: `$ARGUMENTS`

1. 현재 폴더에서 `.md` 파일을 모두 찾는다.
2. `$ARGUMENTS` 언어로 번역.
3. 같은 폴더에 `_<언어>` 접미사 붙여 저장.
   예: README.md → README_en.md
```

호출:
```
/translate english
/translate japanese
```

## 실전 스킬 예시

### `/cleanup-downloads` — 다운로드 폴더 정리
```markdown
---
name: cleanup-downloads
description: 다운로드 폴더 자동 분류
---

# Downloads 정리

1. ~/Downloads 안의 파일을 분류:
   - 이미지 (.jpg/.png/.gif) → ~/Downloads/images/
   - 문서 (.pdf/.docx) → ~/Downloads/docs/
   - 영수증 (영수/receipt 키워드 포함) → ~/Downloads/receipts/
   - 코드 (.zip/.tar.gz with src/) → ~/Downloads/code/

2. 30일 지난 파일은 archive/ 로 추가 이동.
3. 이동 전 미리보기 후 사용자 확인.
```

### `/git-clean` — 안전한 커밋
```markdown
---
name: git-clean
description: 변경사항 검토 + 안전한 커밋 + 푸시
---

# Git 안전 커밋

1. `git status`로 변경 파일 확인.
2. 변경된 파일에 .env, *.key 같은 민감 파일이 있는지 검사. 있으면 중단.
3. `git diff`로 사용자에게 변경 미리보기.
4. 적절한 커밋 메시지 자동 생성.
5. 사용자 승인 후 commit + push.
```

### `/blog-post` — 블로그 초안
```markdown
---
name: blog-post
description: 주제를 받아 블로그 글 초안 작성 ($ARGUMENTS = 주제)
---

# 블로그 초안

1. `$ARGUMENTS` 주제에 대한 블로그 글 초안 작성.
2. 구조:
   - 제목 (한국어 + 영어 슬러그)
   - 1줄 요약
   - 도입 (3-5줄)
   - 본문 (3개 섹션)
   - 결론
3. `posts/<날짜>-<슬러그>.mdx` 로 저장.
4. 첫 줄 frontmatter에 date, tags 자동 채움.
```

### `/code-review` — 코드 리뷰
```markdown
---
name: code-review
description: 현재 git diff에 대한 자가 리뷰
---

# 코드 리뷰

1. `git diff main...HEAD` 결과 확인.
2. 다음 관점으로 검토:
   - 보안: 비밀번호/토큰 노출?
   - 성능: 비효율적인 반복?
   - 가독성: 변수명/주석 적절?
   - 테스트: 커버리지 충분?
3. 발견 사항을 review-notes.md 로 정리.
```

## 스킬 디버깅

### `/help` 로 등록 확인

```
/help
```

설치된 스킬이 모두 표시됩니다.

### 안 보이면

1. 폴더 위치 확인 (`~/.claude/skills/<이름>/SKILL.md`)
2. frontmatter 문법 (`---` 위아래로 닫혔는지)
3. `name`, `description` 필드 필수
4. Claude 재시작

## 스킬 vs CLAUDE.md

언제 어느 걸?

| 정보 | 어디에? |
|---|---|
| 이 프로젝트의 규칙 | CLAUDE.md |
| 매번 자동 적용될 컨벤션 | CLAUDE.md |
| **수동으로 호출하는 작업** | **Skills** |
| 한 번에 여러 단계 자동화 | Skills |

> 💡 둘은 보완 관계. CLAUDE.md에 "회의록은 항상 ✅/❓/⚡ 표시" 적고, `/meeting-notes` 스킬에선 그 규칙을 자동 적용.

## 팀과 공유

스킬을 git에 커밋하면 팀원도 같은 매크로 사용 가능.

```
프로젝트/
├── .claude/
│   └── skills/
│       ├── deploy/
│       │   └── SKILL.md
│       └── release-notes/
│           └── SKILL.md
└── ...
```

이렇게 두면 프로젝트별 스킬이 됩니다.

## 자주 묻는 질문

<details>
<summary><b>Q. 스킬에서 외부 명령 실행할 수 있어요?</b></summary>

스킬 내용에 "X 명령을 실행" 이라고 적으면 Claude가 그 명령을 실행하려고 시도해요. 권한 시스템은 그대로 적용.
</details>

<details>
<summary><b>Q. 스킬이 너무 많아지면?</b></summary>

각 스킬을 명확한 카테고리(폴더)로 묶거나, 자주 안 쓰는 건 비활성화. `~/.claude/skills/.disabled/` 같은 식.
</details>

<details>
<summary><b>Q. 스킬이 자동으로 실행되게 하려면?</b></summary>

Skills는 **사용자 명시 호출** 전용입니다. 자동 실행은 [Hooks](/advanced/hooks) 영역.
</details>
