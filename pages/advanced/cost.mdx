# 20 · 비용 절약 팁

> "한도 다 써서 또 답답해..." — 이 페이지를 읽으면 같은 비용으로 2배 더 쓸 수 있어요.

## 비용 = 토큰 = 컨텍스트

핵심 공식 하나만 기억하세요.

> **컨텍스트에 들어간 모든 글자 × 모델 단가 = 이번 응답 비용**

토큰을 적게 쓰는 습관 = 비용 절약.

## 절약 핵심 10가지

### 1. 작은 모델로 충분한 일은 작은 모델로

| 작업 종류 | 추천 모델 |
|---|---|
| 단순 정리, 번역, 요약 | **Haiku** (가장 저렴) |
| 일반적인 코드 작성, 분석 | **Sonnet** |
| 복잡한 추론, 어려운 디버그 | **Opus** |

```
/model
```

으로 빠르게 전환.

### 2. 작업 단위로 `/clear`

같은 세션을 며칠 끌면 컨텍스트가 누적돼서 매 응답마다 그 누적분이 다시 보내집니다.

```
작업 끝 → /clear → 새 작업
```

### 3. CLAUDE.md 짧게 유지

CLAUDE.md는 **매 세션 시작 시 항상 읽힙니다.** 500줄이면 매번 500줄어치 토큰.

- 200줄 안쪽 추천
- 디테일은 별도 파일 → `@file.md` 로 필요할 때만 가져오기

### 4. 큰 파일은 부분만 읽기

```
> file.pdf 통째로 읽지 말고, 처음 5페이지만 읽고 요약해줘
```

또는

```
> grep -n "Error" log.txt 결과만 봐줘
```

### 5. 출력 길이 제한

```
> 결론만 3줄로
> 표 형식으로, 각 셀은 10자 이내
> 키워드만 나열
```

응답이 짧으면 출력 토큰도 적게.

### 6. 자주 쓰는 명령은 settings allow

매번 권한 묻는 것도 토큰 소모. `.claude/settings.json` 의 `allow` 에 자주 쓰는 명령 등록.

### 7. 서브에이전트 적절히

검색 같은 일은 서브에이전트가 따로 처리 → 메인 컨텍스트에 부담 안 줌.

```
> 코드베이스 검색은 Explore 에이전트에게.
```

### 8. `/compact` 로 다이어트

긴 대화는 핵심만 압축:

```
/compact "최근 코드 변경만 자세히, 잡담은 줄여서"
```

### 9. 결과는 파일로

긴 산출물을 다음 세션이 알게 하려면:

```
> 분석 결과를 analysis.md 로 저장 (다음 세션에서 다시 안 만들도록)
```

### 10. `/cost` 자주 확인

```
/cost
```

본인 패턴 어디서 토큰을 많이 쓰는지 인지. 비싼 패턴이 보이면 교정.

## 비용 폭증 패턴 (피해야 할 것)

### ⚠️ 패턴 1 · 한 번에 100개 파일

```
> 이 폴더의 100개 PDF 다 읽고 요약해줘
```
→ 컨텍스트 폭발. 토큰 한도 광속 소모.

**대안**:
```
> 폴더 안 PDF 목록만 먼저 확인. 그 다음 1개씩 읽어서 요약하고
   summary 파일에 누적해줘. 메모리 문제 안 나게.
```

### ⚠️ 패턴 2 · 무한 루프

CLAUDE.md / 후크가 이상해서 **자기 자신을 부르는 루프** — 사고형 토큰 소모. 후크 만들 때 조심.

### ⚠️ 패턴 3 · 답이 만족스럽지 않다고 비슷한 질문 반복

```
> 다시
> 이번엔 더 잘
> 한 번 더
```

이러면 매번 전체 컨텍스트가 다시 처리됨. **무엇이 만족스럽지 않은지 명확히** + 한 번에 풍부하게 부탁.

### ⚠️ 패턴 4 · Opus로 단순 작업

"이 폴더에 뭐 있어?" 같은 단순 질문에 Opus 쓰면 낭비. Haiku로 충분.

## 한도 도달 시 대응

### 즉시 할 일
1. `/clear` 또는 새 세션
2. 가벼운 작업으로 전환 (Haiku)
3. 다음 한도 리셋 시간 확인

### 자주 도달하면
- Pro → Max 5x 업그레이드 고려
- 작업 분할 / 일정 분산

## 비용 모니터링 자동화

### 후크로 자동 로깅

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [{
          "type": "command",
          "command": "echo \"$(date) - session ended\" >> ~/.claude/usage.log"
        }]
      }
    ]
  }
}
```

### 일/주 단위 리포트

스킬 하나 만들기:

```markdown
---
name: usage-report
description: 이번 주 토큰 사용 패턴 보고
---

1. ~/.claude/usage.log 읽기
2. 일/세션별 통계
3. 가장 비싼 세션 Top 3
4. 절약 제안
```
