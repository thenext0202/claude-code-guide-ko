# 15 · MCP — 외부 도구 연결

## MCP가 뭔가요?

> **MCP (Model Context Protocol) = AI가 외부 서비스와 대화하는 표준 콘센트**

AI는 기본적으로 자기 컴퓨터(=내 폴더) 안에서만 일해요. 그런데 **GitHub, Slack, Notion, DB**처럼 바깥 서비스를 만지려면 어떻게 할까요?

→ **MCP 서버**를 꽂으면 됩니다. 콘센트(=MCP 클라이언트, Claude Code)에 플러그(=MCP 서버, 각 서비스 연결기)를 꼽는 셈.

## 어떤 게 가능한가요?

| MCP 서버 | 무엇을 시킬 수 있나? |
|---|---|
| **GitHub** | "이 PR에 리뷰 코멘트 달아줘", "최근 이슈 50개 정리" |
| **Slack** | "#dev 채널 어제 대화 요약", "팀에 알림 보내기" |
| **Notion** | "이 회의록 노션 페이지로 발행" |
| **Linear/Jira** | "내 티켓 목록 가져와서 우선순위 정리" |
| **PostgreSQL/MySQL** | "사용자 테이블 통계 쿼리" |
| **Figma** | "이 디자인 컴포넌트 React로 변환" |
| **Google Drive** | "어제 받은 PDF 요약" |
| **Filesystem** | 다른 폴더 접근 (sandbox 확장) |
| **Brave Search** | 일반 웹 검색 |
| **Puppeteer** | 브라우저 자동화 |

> 💡 거의 모든 SaaS는 MCP 서버가 이미 있거나 만들 수 있어요.

## 어떻게 추가해요?

### 방법 1 · `/mcp` 명령

```
/mcp
```

설치 가능한 MCP 서버 목록과 현재 연결 상태 표시.

### 방법 2 · `claude mcp add` 명령

터미널에서 직접:

```bash
claude mcp add github
```

또는 임의의 서버:

```bash
claude mcp add my-server -- node /path/to/server.js
```

### 방법 3 · 설정 파일

`.claude/mcp.json` 또는 `~/.claude/mcp.json` 에:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "your_token_here"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://user:pass@localhost/mydb"]
    }
  }
}
```

> ⚠️ 토큰/비밀번호 적을 때는 **환경 변수**로 빼는 게 더 안전.

## 실전 예시 — GitHub MCP

### 셋업

1. GitHub Personal Access Token 발급 ([Settings → Developer → Tokens](https://github.com/settings/tokens))
2. 권한: `repo`, `read:user`
3. 환경변수 또는 mcp.json에 등록
4. Claude 재시작

### 사용

```
> @github 이 레포에서 최근 닫힌 이슈 10개 보여줘.
   라벨별로 그룹핑하고, 평균 닫힘 시간도 계산해줘.
```

```
> @github 'feature/dark-mode' 브랜치로 PR 만들어줘.
   제목: '다크모드 지원'
   본문: 변경된 파일 목록 + 자동 생성된 설명
```

## 실전 예시 — Slack MCP

```
> @slack #engineering 채널 오늘 메시지 요약해줘.
   결정 사항만 따로 뽑아서.
```

```
> @slack 빌드 끝났다고 #release 채널에 알려줘.
   메시지: "🚢 v2.3.0 production 배포 완료"
```

## 실전 예시 — DB MCP

```
> @postgres 어제 가입한 유저 수 알려줘.
   국가별로 그룹핑해서 표로.
```

> ⚠️ 운영 DB 직접 연결은 위험. **읽기 전용 권한**으로 분리하는 게 안전.

## MCP 만들기 (개발자용)

표준 프로토콜이라 직접 만들 수도 있어요.

### 최소 예시 (Node.js)

```javascript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";

const server = new Server({
  name: "my-tool",
  version: "0.1.0"
});

server.setRequestHandler("tools/list", async () => ({
  tools: [{
    name: "say_hello",
    description: "Say hello to someone",
    inputSchema: {
      type: "object",
      properties: { name: { type: "string" } }
    }
  }]
}));

server.setRequestHandler("tools/call", async (req) => {
  if (req.params.name === "say_hello") {
    return {
      content: [{ type: "text", text: `Hello, ${req.params.arguments.name}!` }]
    };
  }
});

server.start();
```

자세한 건 [MCP 공식 문서](https://modelcontextprotocol.io).

## MCP vs Plugins vs Skills

자주 헷갈리는 셋의 차이.

| 개념 | 정의 | 어디서 만드나? |
|---|---|---|
| **Plugin** | Claude Code 자체 확장 (UI/명령어 추가) | 플러그인 마켓플레이스 |
| **MCP** | 외부 서비스 연결기 | 표준 MCP 프로토콜 |
| **Skill** | 반복 작업 자동화 레시피 | 마크다운 + 메타데이터 |

대충: **Plugin = 어플 확장, MCP = 외부 콘센트, Skill = 매크로**

## 보안 주의

- **신뢰할 수 있는 MCP 서버**만 사용. 코드 검토 후 도입
- 비밀번호/토큰은 환경변수, 절대 git에 푸시 금지
- 운영 DB는 읽기 전용으로 분리
- MCP 서버에 권한을 너무 넓게 주지 않기 (필요한 것만)

## 자주 묻는 질문

<details>
<summary><b>Q. MCP 서버가 멈췄는지 어떻게 알아요?</b></summary>

`/mcp` 로 상태 확인. `disconnected` 면 재시작.
</details>

<details>
<summary><b>Q. 한국 서비스 (카카오, 네이버) MCP는?</b></summary>

공식은 적습니다. 직접 만들거나, 해당 서비스 API + 일반 도구로 우회.
</details>

<details>
<summary><b>Q. MCP가 너무 많으면 느려져요?</b></summary>

각 서버는 별도 프로세스라 직접적으론 영향 적지만, **컨텍스트는 공유**해서 도구 정의가 많을수록 토큰 소모. 안 쓰는 건 비활성화 권장.
</details>
