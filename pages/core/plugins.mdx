# 06 · 플러그인 사용법

## 플러그인이 뭐예요?

> **플러그인 = Claude Code에 새로운 기능을 더해주는 작은 모듈**

기본 Claude Code도 강력하지만, 특정 작업(예: PDF 처리, 디자인 자동화, 번역)에 특화된 도움이 필요할 때 플러그인을 깝니다.

비유하자면 — **브라우저(크롬)에 광고 차단 확장 프로그램을 설치하는 것**과 비슷해요.

## 어떻게 깔아요?

### 방법 1 · `/plugins` 명령어

Claude를 실행한 상태에서:

```
/plugins
```

설치 가능한 플러그인 목록이 나옵니다. 화살표 키로 골라서 엔터.

### 방법 2 · 마켓플레이스 추가 후 설치

특정 마켓플레이스(플러그인 저장소)에서 받으려면:

```
/plugin marketplace add <마켓플레이스-주소>
/plugin install <플러그인-이름>
```

### 방법 3 · 설정 파일로 미리 등록

`~/.claude/settings.json` 또는 프로젝트의 `.claude/settings.json`에:

```json
{
  "plugins": {
    "marketplaces": ["https://example.com/my-marketplace"],
    "enabled": ["plugin-name"]
  }
}
```

이렇게 하면 다음 `claude` 실행 시 자동 활성화돼요.

## 추천 플러그인 (입문자용)

> ⚠️ 플러그인 생태계는 빠르게 변합니다. 아래는 대표 카테고리만 소개. 최신 목록은 `/plugins` 로 직접 확인하세요.

### 📝 문서·번역 도우미
- 문서 일괄 번역, 한·영 동시 작성을 단축어로 처리

### 🐛 디버깅 도우미
- 에러 로그를 자동 분석해서 원인 추정 + 수정 제안

### 📊 데이터 정제
- 엑셀/CSV 클렌징, 중복 제거, 표준화 포맷 변환

### 🎨 디자인·이미지
- 로고 생성, 이미지 일괄 리사이즈, 썸네일 자동화

### 🔍 코드 리뷰
- PR 단위로 코드 리뷰 코멘트 자동 작성

## 만들어진 플러그인이 마음에 안 든다면?

직접 만들 수도 있어요. 플러그인은 결국 **마크다운 + 설정 파일** 묶음이라, [17 Skills](/advanced/skills)와 컨셉이 비슷해요. 사실상 Skills로 본인만의 작업 세트를 만드는 게 더 간단할 때가 많습니다.

## 자주 듣는 질문

<details>
<summary><b>Q. 플러그인이 안 보여요</b></summary>

`/plugin marketplace list` 로 마켓플레이스가 등록됐는지 확인. 새 마켓플레이스 추가 후엔 Claude를 한 번 재시작해보세요.
</details>

<details>
<summary><b>Q. 플러그인을 끄고 싶어요</b></summary>

`/plugins` 메뉴에서 disable 선택. 또는 `settings.json`에서 `enabled` 배열에서 제거.
</details>

<details>
<summary><b>Q. 한국어 플러그인이 부족한데요</b></summary>

지금은 영어권 플러그인이 더 많아요. 그래서 [17 Skills](/advanced/skills) 로 직접 한국어 작업 자동화를 만드는 게 실용적입니다.
</details>
