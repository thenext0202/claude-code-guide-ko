# 10 · 세션 관리

> "Claude가 갑자기 똑똑하지 않아요" — 이 문제의 80%는 세션 관리 문제입니다.

> 📚 **이 페이지 한 눈에**
> - **세션** = `claude` 켜서 끌 때까지 한 번의 대화 단위
> - 컨텍스트는 시간이 지나면 누적 → 답변 품질 저하 → `/clear` 또는 새 세션
> - **`/clear`**: 대화 완전 비움 / **`/compact`**: 요약해서 유지
> - 작업 단위로 새 세션 시작이 핵심 습관
>
> 🔗 함께 보면 좋은 페이지: [📖 용어집 — 컨텍스트](/extras/glossary#컨텍스트-context) · [07 핵심 개념](/core/concepts) · [20 비용 절약](/advanced/cost)

## "세션"이 뭐예요?

> **세션 = `claude` 명령으로 켜진 한 번의 대화 단위**

세션이 끝나면 Claude는 그 대화를 잊어요(아주 짧게는 기억하지만). 그래서 **세션을 잘 끊고 잘 이어붙이는 것**이 효율의 핵심입니다.

## 세션의 3가지 상태

```
🟢 새 세션          → 컨텍스트 깨끗, 똑똑함 100%
🟡 진행 중          → 컨텍스트 차오름, 똑똑함 점점 ↓
🔴 컨텍스트 거의 참 → 답변 품질 저하, 메모리 누락 가능
```

## 세션 잘 쓰는 5가지 패턴

### 패턴 1 · 작업당 새 세션

가장 중요. **작업이 바뀌면 `/clear`** 또는 **재실행**.

```
1. 회의록 정리 작업 시작
2. (작업 완료)
3. /clear
4. "이번엔 영수증 폴더 정리할게..."
```

이전 작업의 컨텍스트가 새 작업을 흐리지 않게.

### 패턴 2 · 큰 작업은 단계로

큰 프로젝트는 세션을 통째로 잡지 말고 **단계별로 끊으세요.**

```
[세션 1] 요구사항 정리 → CLAUDE.md 업데이트 → /exit
[세션 2] 핵심 모듈 구현 → /exit
[세션 3] 테스트 작성 → /exit
[세션 4] 리팩토링 → /exit
```

각 세션 사이에 CLAUDE.md / 메모리에 진행 상황을 남겨두면, 다음 세션이 매끄러워요.

### 패턴 3 · `/compact` 로 살리기

컨텍스트가 차서 끊기 아까울 때.

```
/compact "지금까지 한 코드 변경만 자세히 남기고 잡담은 줄여줘"
```

핵심만 남고 나머진 압축. 좀 더 길게 갈 수 있어요.

> ⚠️ `/compact`도 한 번에 압축할 수 있는 양에 한계가 있어요. 그래도 너무 길면 결국 새 세션이 답.

### 패턴 4 · 결과를 파일로 저장

긴 대화 내용을 다음 세션이 알 수 있게 하려면 **파일로 출력해두세요.**

```
> 우리가 결정한 사항을 docs/decisions.md 에 정리해줘.
```

다음 세션에서:

```
> docs/decisions.md 읽어보고 이어서 진행해줘.
```

### 패턴 5 · 메모리 활용

장기적으로 기억해야 할 사항은 [auto memory 시스템](/advanced/workspace)이나 CLAUDE.md에 저장. 세션 간에 자동 유지.

## "어, 갑자기 멍청해진 것 같아"

이 증상의 진단표.

| 증상 | 원인 가능성 | 해결 |
|---|---|---|
| 같은 실수 반복 | 컨텍스트 누적 | `/clear` |
| 파일 위치 자꾸 헷갈림 | CLAUDE.md 부족 | CLAUDE.md 업데이트 |
| 오래된 정보 기준으로 답변 | 옛 세션 정보 잔존 | 새 세션 |
| 답변이 점점 짧아짐 | 컨텍스트 거의 참 | `/compact` 또는 새 세션 |
| 모델 자체가 별로인 듯 | 모델 선택 잘못 | `/model` 로 Opus 등 |

## 컨텍스트 사용량 체크

```
/cost
```

위 명령어로 토큰 소모량 확인.

또는 화면 아래쪽 상태바에 보통 진행률이 표시됩니다.

```
[Context: 45% used]
```

70% 넘기 시작하면 마무리 짓고 새 세션 준비.

## 멀티 세션 (동시에 여러 개)

여러 작업을 병렬로 하고 싶으면 **터미널 창을 여러 개** 열면 됩니다. 각 창마다 독립된 Claude 세션이 돌아요.

```
터미널 1: 사이트 코드 작업 폴더에서 claude
터미널 2: 회의록 폴더에서 claude
터미널 3: 데이터 분석 폴더에서 claude
```

각 세션은 서로 모릅니다. 같은 폴더에서 같은 작업하면 충돌 가능하니 주의.

## 세션 기록은 어디에?

Claude Code는 세션 로그를 로컬에 저장해요. 보통:

- macOS/Linux: `~/.claude/projects/<프로젝트해시>/`
- Windows: `C:\Users\<이름>\.claude\projects\<프로젝트해시>\`

> 🛡 이 로그에 민감 정보가 저장될 수 있으니, 회사 컴퓨터라면 회사 보안 정책 확인.

## 자주 묻는 질문

<details>
<summary><b>Q. 어제 하던 거 이어서 할 수 있나요?</b></summary>

기본은 안 됩니다 (새 세션 = 새 컨텍스트). 다만 어제 결과를 파일/CLAUDE.md/메모리에 저장해두면 새 세션에서 다시 읽고 이어갈 수 있어요.

`claude --resume` 으로 이전 세션을 재개하는 방법도 있습니다 (사용 가능 여부는 환경마다).
</details>

<details>
<summary><b>Q. 한 세션을 며칠 동안 켜놔도 돼요?</b></summary>

기술적으론 가능. 권장하진 않아요 — 컨텍스트가 점점 누적돼서 품질 저하. 길어야 한 작업, 짧으면 한 시간 단위로 끊어 쓰세요.
</details>

<details>
<summary><b>Q. /compact 와 /clear 차이?</b></summary>

- `/compact`: 대화를 **요약해서 유지** (어제 한 일 일부 기억)
- `/clear`: 대화를 **완전히 비움** (백지)
</details>
