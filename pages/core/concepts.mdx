# 07 · 핵심 개념 (Context · Tools · Permissions)

> 이 페이지를 이해하면, **Claude Code의 모든 동작이 한꺼번에 이해됩니다.** 진짜로요.

> 📚 **이 페이지 한 눈에**
> - **Context**: 지금 이 순간 Claude가 알고 있는 모든 것 (대화 + 파일 + 설정)
> - **Tools**: Claude가 컴퓨터에 손대는 능력들 (Read·Write·Bash·Edit 등)
> - **Permissions**: 위험한 작업 전 묻기·자동 허용·자동 차단 규칙
>
> 🔗 함께 보면 좋은 페이지: [📖 용어집](/extras/glossary) · [08 CLAUDE.md](/core/claude-md) · [10 세션 관리](/core/sessions) · [16 Hooks](/advanced/hooks)

세 가지만 기억하세요.

## 1️⃣ Context (컨텍스트) — AI의 작업 메모리

> **컨텍스트 = 지금 이 순간 Claude가 알고 있는 모든 것**

Claude는 한 번에 무한정 많은 걸 기억할 수 없어요. 한 번의 작업에서 다룰 수 있는 양이 정해져 있는데, 이걸 **컨텍스트 윈도우** 라고 해요.

### 컨텍스트에 들어가는 것
- 여러분의 질문/대화 내용
- 읽은 파일들
- 시스템 지시문 (CLAUDE.md 등)
- 도구 사용 결과

### 왜 알아야 하나요?

큰 파일 100개를 한 번에 시키면 → **컨텍스트 초과** → 일부 잊어버림 → 결과 품질 떨어짐.

### 실전 팁

| 좋은 패턴 | 나쁜 패턴 |
|---|---|
| 작업 단위로 쪼개서 부탁 | "이 모든 파일을 한꺼번에 분석해" |
| 끝난 작업 후 `/clear` | 며칠 동안 한 세션 계속 사용 |
| 큰 파일은 부분만 읽게 | 전체 PDF 100페이지 통째로 |

## 2️⃣ Tools (도구) — Claude가 쓸 줄 아는 능력들

> **Tools = Claude가 컴퓨터에 손을 댈 수 있는 방법들**

기본 Claude는 글만 쓸 수 있지만, Claude Code는 다양한 도구를 가지고 있어서 **실제 작업**을 합니다.

### 기본 도구들

| 도구 | 무엇을 하나요? |
|---|---|
| **Read** | 파일을 읽음 |
| **Write** | 새 파일 만들기 |
| **Edit** | 기존 파일 수정 |
| **Bash** | 터미널 명령어 실행 |
| **Glob** | 패턴으로 파일 찾기 |
| **Grep** | 파일 내용 검색 |
| **WebFetch** | 웹 페이지 가져오기 |
| **WebSearch** | 인터넷 검색 |

### 확장 도구 (MCP)

- Slack 메시지 보내기
- GitHub PR 만들기
- DB 쿼리 실행
- Figma 디자인 가져오기
- ... 무한 확장 가능

[15 MCP](/advanced/mcp)에서 자세히.

### 관찰 포인트

대화 중에 이런 표시가 보일 거예요:

```
✻ Reading hello.txt... (Read tool)
✻ Editing main.py at line 42... (Edit tool)
✻ Running npm install... (Bash tool)
```

**어떤 도구를 쓰는지 보면, AI가 뭘 하고 있는지 알 수 있어요.**

## 3️⃣ Permissions (권한) — 안전벨트

> **Permissions = "이건 해도 돼", "이건 물어봐" 같은 규칙**

Claude는 함부로 파일을 지우거나 큰 변경을 하지 않아요. 위험한 작업이면 항상 **확인**을 먼저 받습니다.

### 권한 모드

| 모드 | 동작 |
|---|---|
| **default** (기본) | 위험한 도구는 매번 Yes/No 묻기 |
| **acceptEdits** | 편집은 자동 허용, 명령 실행은 묻기 |
| **plan** | 아무것도 안 바꾸고 계획만 보여줌 |
| **bypassPermissions** | 거의 다 자동 허용 (⚠️ 위험) |

`/permissions` 명령으로 언제든 변경 가능.

### 권한 화면 예시

```
🔒 Tool: Bash
   Command: rm -rf old_folder

   1. Yes
   2. Yes, and don't ask again for similar
   3. No, and tell Claude what to do differently
```

여기서 **무엇을 한다는지 한 번 읽고** 결정하세요.

### 미리 허용하기

자주 쓰는 명령은 `.claude/settings.json` 에 미리 등록:

```json
{
  "permissions": {
    "allow": [
      "Bash(npm install)",
      "Bash(git status)",
      "Read(**)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Edit(.env)"
    ]
  }
}
```

## 셋이 어떻게 협력하나요?

```
사용자 부탁 입력
   ↓
[Context] 현재 대화 + 파일 내용 + CLAUDE.md 종합
   ↓
Claude가 어떤 [Tools]를 쓸지 결정
   ↓
[Permissions] 확인 (Yes/No)
   ↓
도구 실행 → 결과 다시 [Context]에 추가
   ↓
다음 단계 또는 응답
```

이 흐름을 머릿속에 그릴 수 있으면, **AI가 뭔가 이상하게 동작할 때 어디가 문제인지 짚어낼 수 있어요.**

## 자주 헷갈리는 것들

<details>
<summary><b>Q. 컨텍스트가 꽉 찼다는 메시지가 나와요</b></summary>

`/clear` 로 대화 비우거나, 새 세션 시작. 또는 `/compact` 로 이전 대화 압축. ([10 세션 관리](/core/sessions) 참고)
</details>

<details>
<summary><b>Q. 권한 묻는 게 너무 많아요. 시끄러워요</b></summary>

`/permissions` → `acceptEdits` 모드로 전환. 또는 자주 쓰는 명령을 settings.json에 allow 등록.
</details>

<details>
<summary><b>Q. Claude가 갑자기 멍청해진 것 같아요</b></summary>

대부분 **컨텍스트 누적** 문제예요. 새 세션 만들면 회복됩니다.
</details>
