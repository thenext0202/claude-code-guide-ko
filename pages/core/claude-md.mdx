# 08 · CLAUDE.md — AI에게 장기 기억 주기

> "매번 똑같은 거 또 설명해야 해? 너무 귀찮아."

이 문제를 해결해주는 파일이 **CLAUDE.md** 입니다. 한 번 써두면 Claude가 매 대화마다 자동으로 읽어요.

> 📚 **이 페이지 한 눈에**
> - 폴더에 `CLAUDE.md` 두면 **모든 세션에 자동 적용** (신입사원 매뉴얼 같은 것)
> - 위치 3가지: `./CLAUDE.md`(프로젝트), `~/.claude/CLAUDE.md`(전역), `./CLAUDE.local.md`(개인)
> - **`/init`** 명령으로 자동 생성 → 다듬기
> - 적기 좋은 것: 프로젝트 개요, 코드 스타일, 자주 쓰는 명령, 절대 금지사항
> - 적지 말 것: 비밀번호, 매일 바뀌는 정보
>
> 🔗 함께 보면 좋은 페이지: [📖 용어집](/extras/glossary#claudemd) · [✏️ 에디터 셋업](/extras/editor-setup#5-claudemd-만들기) · [07 핵심 개념](/core/concepts) · [19 워크스페이스 설계](/advanced/workspace)

## CLAUDE.md가 뭐예요?

> **CLAUDE.md = 이 폴더에 들어오는 Claude에게 매번 자동으로 읽히는 안내서**

폴더에 이 이름의 마크다운 파일을 두면, Claude는 작업 시작할 때 자동으로 읽고 그 내용을 기억해요.

비유하자면 **신입사원에게 주는 회사 매뉴얼** 같은 것.

## 어디에 만들어요?

세 군데에 만들 수 있어요. **위에 있을수록 우선순위 높음.**

| 위치 | 무엇을 적나? | 적용 범위 |
|---|---|---|
| **`./CLAUDE.md`** (프로젝트 폴더) | 이 프로젝트만의 규칙 | 이 폴더에서 실행할 때 |
| **`~/.claude/CLAUDE.md`** (홈 폴더) | 내 개인 선호 | 모든 작업 |
| **`./CLAUDE.local.md`** | 깃에 올리지 않는 개인 메모 | 이 프로젝트, 나만 |

## 어떻게 시작하나요?

### 방법 1 · `/init` 명령 (가장 쉬움)

새 폴더에서 Claude를 켠 다음:

```
/init
```

이러면 Claude가 폴더 내용을 분석해서 **CLAUDE.md 초안을 자동 생성**해줍니다. 그걸 본인 입맛에 맞게 다듬으면 끝.

### 방법 2 · 직접 작성

좋아하는 에디터로 `CLAUDE.md` 파일을 만들고 자유롭게 작성. 마크다운 문법.

## 무엇을 적어야 잘 적은 건가요?

### ✅ 적으면 좋은 것들

```markdown
# 프로젝트 개요
이 폴더는 한국 카페 추천 사이트의 코드입니다.
- 기술: Next.js 14 + Tailwind
- 배포: Railway

# 코드 스타일
- 함수명은 영어 camelCase
- 주석은 한국어로

# 자주 쓰는 명령어
- 개발 서버: `npm run dev`
- 배포 빌드: `npm run build`
- 테스트: `npm test`

# 절대 하지 마세요
- `.env` 파일 수정 금지
- main 브랜치에 직접 푸시 금지

# 자주 묻는 폴더 구조
- pages/ : 라우트
- components/ : 재사용 컴포넌트
- lib/db.ts : DB 연결 설정 (수정 시 신중)
```

### ❌ 적지 말아야 할 것

- 비밀번호, API 키 (절대!)
- 너무 시시콜콜한 디테일 — 짧고 중요한 것만
- 매일 바뀌는 정보 — 자주 업데이트 못 하면 오히려 혼란

## 비개발자용 CLAUDE.md 예시

코딩 폴더가 아니어도 좋아요.

```markdown
# 이 폴더는?
"마케팅 자료" 라는 이름의 폴더.
회의록, 광고 시안, 캠페인 보고서가 모여있음.

# 파일 명명 규칙
- 회의록: meeting_YYYY-MM-DD.md
- 광고 시안: ad_캠페인이름_v버전.psd
- 보고서: report_YYYY-MM_주제.docx

# 정리 원칙
- 매주 금요일 정리 시간
- 1년 지난 자료는 archive/ 폴더로
- 회의록은 영문 키워드 태그 추가

# 회의록 작성 시
- 첫 줄에 참석자 명단
- 결정사항은 ✅
- 미정 사항은 ❓
- 액션 아이템은 ⚡ + 담당자 이름
```

## 실전 효과

### Before (CLAUDE.md 없을 때)
```
> 회의록 정리해줘
✻ 어떤 형식으로 정리할까요? 참석자 명단을 어떻게...
> 첫 줄에 참석자, 결정은 ✅, ...
✻ 알겠습니다. 작업하겠습니다.
```

### After (CLAUDE.md 있을 때)
```
> 회의록 정리해줘
✻ 회의록 정리 시 규칙 (CLAUDE.md 참조):
   ✅ 결정사항, ❓ 미정, ⚡ 액션 아이템
   바로 작업 시작합니다.
```

같은 부탁이지만 **반복 설명이 사라집니다.**

## 고급 팁

### 여러 CLAUDE.md를 계층적으로

큰 프로젝트는 폴더마다 두는 게 좋아요.

```
프로젝트/
├── CLAUDE.md             ← 전체 규칙
├── frontend/
│   └── CLAUDE.md         ← 프론트엔드만의 규칙
└── backend/
    └── CLAUDE.md         ← 백엔드만의 규칙
```

Claude는 작업 위치에 따라 가장 가까운 것부터 위로 올라가며 읽어요.

### 메모리 시스템과 함께

CLAUDE.md는 **고정된 정보** 용. 학습되는 정보(예: "이 사용자는 함수형 스타일을 선호") 는 [auto-memory 시스템](/core/sessions)을 별도로 활용.

### "@" 로 다른 파일 가져오기

```markdown
# 코드 컨벤션
자세한 건 @docs/style-guide.md 를 참조하세요.
```

이렇게 적으면 Claude가 그 파일도 같이 읽어줍니다.

## 자주 묻는 질문

<details>
<summary><b>Q. CLAUDE.md 너무 길어지면 안 좋나요?</b></summary>

토큰을 더 쓰게 되긴 해요. 하지만 200~500줄 정도까지는 문제 없습니다. 너무 길어지면 핵심만 두고 상세는 별도 파일로 분리(@import).
</details>

<details>
<summary><b>Q. 팀원들과 같이 쓰면 좋겠는데?</b></summary>

`CLAUDE.md`는 Git에 올리세요. 팀 공통 규칙이 됩니다. 본인만의 개인 메모는 `CLAUDE.local.md`로 분리.
</details>

<details>
<summary><b>Q. 자동 갱신할 수 있나요?</b></summary>

네. Claude에게 "지금 대화에서 배운 걸 CLAUDE.md에 추가해줘" 라고 부탁하면 됩니다.
</details>
